# OpenClaw로 Next.js 프로젝트 생성하기  
  
Part 1에서는 왜 직접 블로그를 만들게 되었는지, 그리고 왜 Next.js와 OpenClaw를 선택했는지를 설명했습니다.  
  
이제 실제로 프로젝트를 시작한 과정을 이야기해 보겠습니다.  
  
이 단계는 단순히 명령어를 실행하는 과정이 아니라 **AI 기반 개발 환경을 구축하는 과정**이기도 합니다.  
  
---  
  
# 개발 환경 준비  
  
블로그 프로젝트를 시작하기 전에 몇 가지 기본 환경을 준비해야 합니다.  
  
대표적으로 다음과 같은 도구들이 필요합니다.  
  
- Node.js  
- pnpm 또는 npm  
- Git  
- GitHub  
- Vercel  
- OpenClaw  
  
이 중에서도 특히 중요한 것은 Git과 GitHub입니다.  
  
왜냐하면 이 프로젝트는 **Git 기반으로 모든 콘텐츠를 관리하는 구조**이기 때문입니다.  
  
---  
  
# Git 기반 블로그의 장점  
  
일반적인 블로그 플랫폼은 데이터베이스 기반입니다.  
  
하지만 Next.js 블로그는 보통 다음과 같은 구조를 사용합니다.  
  
Markdown 또는 MDX 파일    
→ Git 저장소    
→ 정적 페이지 생성  
  
이 방식의 장점은 매우 많습니다.  
  
예를 들어  
  
- 콘텐츠 버전 관리 가능  
- GitHub 협업 가능  
- 자동 배포 가능  
- 백업이 매우 쉬움  
  
특히 개발자에게는 매우 익숙한 방식입니다.  
  
---  
  
# OpenClaw 설치  
  
OpenClaw는 AI 기반 개발 도구입니다.  
  
일반적인 AI 챗봇과는 조금 다른 역할을 합니다.  
  
OpenClaw는 다음과 같은 작업을 수행할 수 있습니다.  
  
- 프로젝트 생성  
- 코드 수정  
- 버그 분석  
- 빌드 오류 해결  
- Git 작업 지원  
  
즉 단순한 질문 응답이 아니라 **실제 개발 환경을 직접 조작할 수 있는 AI 도구**입니다.  
  
---  
  
# 첫 번째 명령  
  
프로젝트를 시작하면서 가장 먼저 했던 질문은 이것이었습니다.  
  
"Next.js 블로그 프로젝트를 만들어 달라."  
  
이 요청은 매우 단순해 보이지만 실제로는 다음 작업을 포함합니다.  
  
- Next.js 프로젝트 생성  
- 기본 디렉토리 구조 생성  
- MDX 지원 설정  
- 블로그 라우팅 설정  
- 포스트 폴더 생성  
  
OpenClaw는 이 작업을 매우 빠르게 수행했습니다.  
  
---  
  
# 프로젝트 구조  
  
OpenClaw가 생성한 프로젝트 구조는 다음과 비슷했습니다.  
  
project-root    
 ├ src    
 │ ├ app    
 │ ├ components    
 │ ├ lib    
 │    
 ├ posts    
 ├ public    
 ├ package.json    
  
여기서 중요한 폴더는 다음 세 가지입니다.  
  
- src/app    
- src/components    
- posts    
  
---  
  
# app 폴더  
  
Next.js 13 이후 버전에서는 app 디렉토리를 사용합니다.  
  
이 폴더는 페이지 라우팅을 담당합니다.  
  
예를 들어  
  
app/page.tsx  
  
이 파일은 사이트의 메인 페이지가 됩니다.  
  
---  
  
# posts 폴더  
  
posts 폴더는 블로그 콘텐츠를 저장하는 곳입니다.  
  
예를 들어  
  
posts/my-first-post.mdx  
  
이 파일 하나가 블로그 글 하나가 됩니다.  
  
---  
  
# components 폴더  
  
components 폴더에는 재사용 가능한 UI 컴포넌트가 들어갑니다.  
  
예를 들어  
  
- PostCard  
- CategoryList  
- NavigationBar  
  
같은 것들입니다.  
  
---  
  
# 첫 번째 빌드  
  
프로젝트가 생성된 후 가장 먼저 해야 할 일은 빌드를 실행하는 것입니다.  
  
일반적으로 다음 명령을 사용합니다.  
  
pnpm dev  
  
이 명령은 개발 서버를 실행합니다.  
  
---  
  
# 로컬 서버 실행  
  
개발 서버가 실행되면 보통 다음 주소로 접속할 수 있습니다.  
  
http://localhost:3000  
  
이 주소에서 프로젝트가 정상적으로 실행되는지 확인합니다.  
  
---  
  
# 첫 번째 문제 발생  
  
여기서 예상하지 못한 문제가 발생했습니다.  
  
빌드는 성공했지만  
  
페이지가 제대로 표시되지 않았습니다.  
  
예를 들어  
  
- 포스트 목록이 비어 있음  
- MDX가 렌더링되지 않음  
- 스타일이 깨짐  
  
이런 문제가 나타났습니다.  
  
---  
  
# 원인 분석  
  
문제를 분석해 보니 원인은 MDX 설정이었습니다.  
  
Next.js에서 MDX를 사용하려면 추가 설정이 필요합니다.  
  
예를 들어 다음과 같은 패키지가 필요합니다.  
  
@next/mdx  
  
이 패키지는 Next.js에서 MDX 파일을 읽고 렌더링할 수 있도록 해줍니다.  
  
---  
  
# OpenClaw의 도움  
  
이때 OpenClaw에게 다음과 같이 질문했습니다.  
  
"Next.js에서 MDX가 렌더링되지 않는 이유가 무엇인가."  
  
OpenClaw는 다음 사항을 확인했습니다.  
  
- MDX loader 설정  
- next.config 설정  
- remark 플러그인  
  
그리고 필요한 코드를 자동으로 수정했습니다.  
  
---  
  
# AI 개발의 장점  
  
이 경험을 통해 AI 개발의 큰 장점을 느낄 수 있었습니다.  
  
예전 같으면 다음 과정을 거쳐야 합니다.  
  
- StackOverflow 검색  
- GitHub 이슈 확인  
- 공식 문서 읽기  
- 코드 테스트  
  
하지만 AI 도구를 사용하면  
  
문제 분석과 해결이 훨씬 빨라집니다.  
  
---  
  
# 하지만 AI의 한계도 있다  
  
OpenClaw가 모든 문제를 완벽하게 해결해 주는 것은 아닙니다.  
  
AI가 제안한 코드가 항상 정확한 것은 아닙니다.  
  
그래서 다음 과정이 필요합니다.  
  
- 코드 검증  
- 테스트  
- 수정  
  
즉 AI는 **개발 보조 도구**이지 완전한 자동화 도구는 아닙니다.  
  
---  
  
# GitHub 저장소 생성  
  
프로젝트가 어느 정도 안정되면 다음 단계는 GitHub 저장소를 만드는 것입니다.  
  
GitHub는 다음 역할을 합니다.  
  
- 코드 저장  
- 버전 관리  
- 협업  
- 자동 배포 연결  
  
---  
  
# Git 초기화  
  
프로젝트 루트에서 다음 명령을 실행합니다.  
  
git init  
  
그리고 GitHub 저장소를 연결합니다.  
  
git remote add origin  
  
---  
  
# 첫 번째 커밋  
  
프로젝트의 첫 번째 커밋은 매우 중요합니다.  
  
예를 들어  
  
initial commit: nextjs blog structure  
  
이 커밋은 프로젝트의 시작점을 기록합니다.  
  
---  
  
# 자동 배포 준비  
  
Next.js 블로그의 큰 장점 중 하나는 자동 배포입니다.  
  
GitHub와 Vercel을 연결하면 다음 과정이 자동으로 실행됩니다.  
  
git push    
→ Vercel build    
→ 사이트 배포  
  
즉 코드만 업로드하면 웹사이트가 자동으로 업데이트됩니다.  
  
---  
  
# 이 구조의 장점  
  
이 구조는 매우 강력합니다.  
  
특히 블로그 운영에서 다음 장점이 있습니다.  
  
- 서버 관리 불필요  
- 자동 배포  
- 빠른 페이지 속도  
- SEO 친화적  
  
---  
  
# 하지만 아직 끝이 아니다  
  
Next.js 블로그의 기본 구조는 완성되었습니다.  
  
하지만 실제로 운영하려면 아직 많은 기능이 필요합니다.  
  
예를 들어  
  
- 카테고리 시스템  
- 태그 시스템  
- 포스트 목록  
- RSS 피드  
- SEO 설정  
  
이 기능들을 하나씩 추가해야 합니다.
