## 한줄 요약

오늘의 AI 흐름은 **성능 경쟁이 통제권 경쟁으로 바뀌고 있다**는 문장으로 요약된다. 미국 정부는 국가안보를 이유로 Anthropic의 최신 모델 Fable 5와 Mythos 5에 대한 외국인의 접근을 중단하라고 지시했고, Anthropic은 전 세계 고객의 접근을 갑자기 끊으면서도 판단의 기술적 근거가 충분하지 않다고 반발했다. OpenAI는 TanStack npm 공급망 공격 이후 애플리케이션 인증서를 교체하고 macOS 앱 업데이트 시한을 연장하며, 모델 회사도 일반 소프트웨어 기업과 같은 공급망 위험에 노출된다는 사실을 보여줬다. NVIDIA는 최초의 에이전트형 AI 인프라 benchmark인 AgentPerf 결과를 통해 이제 하드웨어 경쟁도 단일 추론 속도가 아니라 전력당 동시 에이전트 수로 측정되기 시작했다고 강조했다. 연구 현장에서는 judge agent 기반 평가 표준, 합성 데이터의 통계적 유효성, LLM을 이용한 재현성 감사, 로봇의 sim-to-real 도구 조작이 주목받는다.

## 오늘의 핵심 포인트

- 미국 정부의 Anthropic 모델 접근 제한은 frontier model이 일반 클라우드 서비스가 아니라 수출통제 대상 전략 기술로 취급될 수 있음을 보여준다.
- 외국인 직원까지 접근 금지 대상에 포함되면서 글로벌 연구팀, 고객 운영, 모델 안전 검증을 동시에 유지하기 어려운 새로운 거버넌스 문제가 생겼다.
- TanStack 공급망 사고는 모델 안전과 별개로 개발 의존성, CI, 코드 서명, 직원 단말 같은 전통적 보안 경계가 AI 서비스의 신뢰를 좌우함을 드러냈다.
- AgentPerf는 에이전트가 한 작업에 수십~수백 회 모델 호출을 연쇄적으로 수행하는 현실을 반영해 처리량과 전력 효율을 함께 측정한다.
- 최신 연구는 에이전트 평가를 A2A와 MCP 기반 프로토콜로 표준화하고, 합성 데이터와 LLM 감사 결과에 통계적 보증을 붙이는 방향으로 나아간다.

## 산업 뉴스와 제품 흐름

### 1) 미국 정부, Anthropic 최신 모델의 외국인 접근을 막다

Anthropic은 미국 정부가 국가안보 권한을 근거로 Fable 5와 Mythos 5에 대한 모든 외국인의 접근을 중단하라는 수출통제 지시를 내렸다고 밝혔다. 적용 범위는 미국 밖의 고객뿐 아니라 미국 안에서 근무하는 외국 국적의 Anthropic 직원까지 포함한다. 회사는 이를 준수하려면 국적을 기준으로 복잡한 접근 제어를 즉시 구현하기 어렵기 때문에 두 모델을 모든 고객에게 일괄 중단할 수밖에 없다고 설명했다. 다른 Anthropic 모델은 영향을 받지 않는다.

이번 사건은 모델 접근권이 제품 정책을 넘어 국가안보 정책의 직접적인 대상이 됐다는 점에서 중요하다. 첨단 반도체는 오랫동안 수출통제를 받아왔지만, 이제는 학습된 모델과 그 사용 권한 자체가 통제 대상이 될 수 있다. 모델이 사이버보안, 생물학, 군사 분석 같은 영역에서 강해질수록 정부는 API 접근을 전략 기술 이전으로 볼 가능성이 커진다.

그러나 지나치게 넓은 국적 기반 제한은 역효과도 낳는다. 글로벌 연구팀의 내부 검증을 막고, 해외 보안 조직이 방어 목적으로 모델을 사용하는 것까지 중단시키며, 공급자가 모델을 안전하게 운영하는 데 필요한 전문가 접근을 제한할 수 있다. 앞으로 frontier AI 정책의 핵심 쟁점은 단순한 공개·비공개 선택이 아니라 능력별 접근 등급, 감사 가능한 사용자 인증, 목적 제한, 국가 간 상호운용 가능한 통제 체계를 얼마나 정교하게 만들 수 있느냐가 될 것이다.

### 2) OpenAI의 TanStack 대응, AI 기업의 약한 고리는 개발 공급망이다

OpenAI는 TanStack npm 공급망 침해에 대한 대응으로 코드 서명 인증서를 교체하고 macOS 사용자에게 ChatGPT Desktop과 Codex App을 최신 버전으로 업데이트하도록 요청했다. 회사는 이전 인증서로 서명된 악성 소프트웨어가 발견되거나 공식 배포물이 무단 변경됐다는 증거는 없다고 밝혔다. Apple과의 조율에 따라 macOS 업데이트 시한은 6월 26일로 연장됐다.

TanStack의 공식 사후 분석에 따르면 공격자는 GitHub Actions의 신뢰 경계를 악용해 42개 패키지에 걸쳐 84개의 악성 버전을 게시했다. 이 사건의 교훈은 AI 보안이 모델의 jailbreak나 prompt injection에만 머물 수 없다는 점이다. 코딩 에이전트와 연결형 AI는 패키지 관리자를 실행하고, 저장소를 읽고, 자격 증명을 사용하며, 결과물을 서명하고 배포한다. 따라서 일반 개발도구의 공급망 침해가 곧 AI 에이전트의 권한 탈취 경로가 된다.

기업은 에이전트가 설치할 수 있는 의존성을 제한하고, lockfile과 provenance를 검증하며, CI 권한을 최소화하고, 비밀정보를 짧은 수명의 자격 증명으로 바꿔야 한다. 에이전트 시대의 보안은 모델 정렬과 소프트웨어 공급망 보안을 별개의 분야로 다루지 않고 하나의 실행 경계로 묶는 데서 시작한다.

### 3) NVIDIA AgentPerf, 인프라 benchmark가 에이전트 경제성을 측정한다

NVIDIA는 Artificial Analysis의 AgentPerf 첫 공개 결과에서 Blackwell Ultra NVL72가 테스트된 에이전트형 AI workload에서 선두를 기록했고, Hopper 대비 메가와트당 20배 많은 에이전트를 실행했다고 발표했다. 공급자 측 발표이므로 독립적인 후속 검증이 필요하지만, benchmark의 평가 단위 자체가 바뀌었다는 점은 의미가 크다.

전통적인 LLM 인프라 평가는 token per second, 첫 token 지연시간, 단일 요청 비용에 집중했다. 하지만 실제 에이전트는 한 작업을 해결하기 위해 계획, 검색, 도구 호출, 코드 실행, 검증을 반복하며 수십~수백 번의 모델 호출을 이어간다. 이때 중요한 것은 한 번의 답변 속도가 아니라 동시 세션을 얼마나 안정적으로 유지하고, 긴 context와 KV cache를 얼마나 효율적으로 처리하며, 전력과 비용 대비 몇 개의 작업을 끝내느냐다.

AgentPerf 같은 benchmark가 자리 잡으면 모델 선택과 하드웨어 구매도 달라질 수 있다. 가장 빠른 모델보다 전체 workflow의 성공률과 비용이 더 좋은 조합이 선택되고, 데이터센터 설계에서는 GPU 수보다 전력당 완료 작업 수가 중요한 지표가 된다. 에이전트 경쟁은 결국 모델, 런타임, 네트워크, 전력 시스템을 함께 최적화하는 full-stack 경쟁이다.

### 4) OpenAI Economic Research Exchange, 노동시장 논쟁을 실증 연구로 옮긴다

OpenAI는 외부 연구자와 함께 AI의 경제적 영향을 조사하는 Economic Research Exchange를 출범했다. 프로그램은 노동자, 기업, 교육, 공공서비스와 경제 전반에서 AI가 생산성과 업무 구조를 어떻게 바꾸는지에 대한 실증 연구를 지원한다. 제안서는 7월 5일까지 접수하며 방법론적 엄밀성, 실행 가능성, 명확한 연구 일정, 외부 증거에 대한 기여 가능성을 기준으로 평가한다.

AI와 일자리 논쟁은 강한 주장에 비해 신뢰할 수 있는 장기 데이터가 부족하다. 제품 사용 로그는 기업 내부에 있고, 연구자는 실제 업무 변화에 접근하기 어렵다. 반대로 기업이 자체 분석만 공개하면 선택 편향과 이해상충 문제가 생긴다. 외부 연구 협력은 이 간극을 줄일 수 있지만, 데이터 접근 규칙, 결과 공개 독립성, 부정적 결과를 포함한 출판 보장이 핵심이다. 중요한 것은 AI가 일자리를 없애는지 아닌지를 한 문장으로 판정하는 것이 아니라, 어떤 직무의 어떤 작업이 바뀌고 혜택과 비용이 누구에게 돌아가는지를 측정하는 일이다.

## 최신 연구 포커스

### 1) AgentBeats: 에이전트 평가도 에이전트와 표준 프로토콜로 수행한다

AgentBeats는 고정된 LLM 중심 benchmark harness가 다양한 에이전트 구조를 공정하게 평가하기 어렵다고 지적한다. 연구진은 judge agent가 평가를 수행하고 모든 참가자가 A2A로 작업을 관리하며 MCP로 도구에 접근하는 Agentified Agent Assessment를 제안했다. benchmark와 평가 대상이 서로 다른 전용 인터페이스를 구현하는 대신 하나의 표준 프로토콜을 공유해 통합 비용과 테스트·운영 환경의 차이를 줄인다.

5개월간의 공개 경쟁에는 12개 분야에서 298개의 judge agent와 467개의 subject agent가 참여했다. 이 결과는 에이전트 평가가 단일 정답률 표에서 상호운용 가능한 평가 생태계로 확장될 수 있음을 보여준다. 다만 judge agent 자체의 편향과 재현성도 평가해야 하므로, 평가자 버전·도구·실행 로그를 함께 공개하는 체계가 필요하다.

### 2) Valid Inference with Synthetic Data: 합성 데이터에도 통계적 보증이 필요하다

이 논문은 LLM이 만든 silicon sample, LLM-as-a-judge 결과, 생성형 단백질 구조처럼 합성 데이터가 연구와 평가에 빠르게 사용되는 상황을 다룬다. 합성 데이터는 더 많은 실험을 가능하게 하지만 편향, 잡음, 모델 오명세를 그대로 포함할 수 있다. 연구진은 현재 과제와 실제 데이터가 있는 과거 과제들이 적절한 의미에서 교환 가능하다는 task exchangeability 조건을 제안하고, 이 조건 아래 유효한 통계 추론을 수행하는 방법을 개발했다.

핵심 메시지는 합성 데이터를 실제 데이터처럼 취급해서는 안 된다는 것이다. 어떤 역사적 과제와 비교할 수 있는지, 편향이 어느 범위에서 유지되는지, 불확실성을 어떻게 보정하는지를 명시해야 한다. LLM 평가와 사회과학 연구에서 합성 데이터 사용이 늘어날수록 생성 모델의 품질보다 추론 절차의 유효성 보증이 더 중요해진다.

### 3) LLM 기반 자동 재현성 감사, 규모와 정밀도의 간극

사회·행동과학 연구 76편을 대상으로 한 연구는 LLM이 원자료를 재분석해 기존 결과의 재현성을 자동 평가할 수 있는지 조사했다. 7편에서는 유효한 효과크기 추정치를 만들지 못했지만, 나머지 연구에서 원 논문의 효과크기를 Cohen's d ±0.05 범위로 복원한 비율은 41%였다. 정성적 결론의 일치율은 96%였고, 비교 대상 인간 재분석자는 각각 34%와 74%를 기록했다.

수치만 보면 자동화의 가능성이 크지만, 정성적 결론 일치와 정확한 효과크기 복원 사이의 격차를 주의해야 한다. 에이전트가 "주장을 지지한다"는 방향은 맞히면서 실제 효과 규모를 잘못 계산할 수 있다. 따라서 LLM은 연구자를 대체하는 최종 판정자보다 대규모 문헌에서 재검토 대상을 찾는 감사 보조자로 사용하는 것이 현실적이다.

### 4) Mana: 로봇의 관절형 도구 조작을 애니메이션 문제로 바꾼다

Mana는 가위나 집게처럼 내부 관절과 접촉을 함께 제어해야 하는 도구 조작을 다룬다. 연구진은 기능적 affordance를 몇 번의 마우스 클릭으로 지정하고, 절차적으로 생성한 grasp keyframe을 motion planning과 강화학습을 통해 조작 trajectory로 변환한다. 네 종류의 관절형 도구에서 grasping과 in-hand manipulation의 zero-shot sim-to-real transfer를 보고했다.

이 접근은 physical AI에서 데이터 제작 비용을 낮추는 방법을 보여준다. 사람이 모든 궤적을 원격 조작으로 시연하는 대신 핵심 자세만 지정하고 나머지를 시뮬레이션과 학습이 채운다. 로봇 에이전트의 확장은 더 큰 VLA 모델뿐 아니라 저렴하게 행동 데이터를 만들고 실제 환경으로 옮기는 pipeline에 달려 있다.

## 오늘의 관찰

오늘 뉴스와 연구를 관통하는 공통 질문은 **누가 AI 시스템의 행동과 접근을 검증하고 통제할 것인가**다. 정부는 국적과 국가안보를 기준으로 모델 접근권을 제한하고, 공급자는 기술적 근거와 운영 가능성을 문제 삼는다. 소프트웨어 공급망에서는 신뢰받는 패키지와 인증서가 공격 경로가 되고, 인프라에서는 전력당 동시 에이전트 수가 새로운 경제성 지표가 된다. 연구 현장에서는 judge agent, 합성 데이터, 자동 재현성 감사가 평가를 확장하지만 그 평가자와 데이터에도 다시 검증이 필요하다.

AI 거버넌스는 선언문만으로 해결되지 않는다. 모델 접근 정책, 직원 권한, 의존성 provenance, 평가 프로토콜, 통계적 가정, 실행 로그를 시스템 안에 구현해야 한다. 2026년의 경쟁력은 가장 강한 모델뿐 아니라 **통제의 근거를 설명하고, 검증 가능한 방식으로 운영하며, 문제가 생겼을 때 영향 범위를 좁힐 수 있는 능력**에서 갈릴 것이다.
