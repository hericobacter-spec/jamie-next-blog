# AI 에이전트로 여행 경비 관리 웹앱 Money Control 만들기

"여행 경비 관리 앱 하나 있었으면 좋겠다."

이 한마디에서 시작됐다. 해외 출장이든 가족 여행이든, 돈을 쓸 때마다 메모장에 적거나 엑셀에 나중에 몰아서 입력하곤 했다. 실시간 환율도 확인해야 하고, 영수증도 모아야 하고, 나중에 정산할 때 골치가 아팠다. 마땅한 앱이 있나 싶어서 찾아보기도 했는데, 내가 원하는 기능이 다 들어있는 건 없었다.

그래서 AI 에이전트한테 만들어달라고 했다. 결과물은 [Money Control](https://jamie-mac-mini.tail-net/money/)이라는 웹앱. 지금 v1.1까지 개발 완료해서 실제로 쓰고 있다. 과정을 정리해본다.

## 시작은 한 줄이었다

SuperAgent라는 프로젝트 폴더 아래에 `money-control`이라는 서브 폴더를 만들어서 시작했다. AI 에이전트한테 요구사항을 이것저것 던졌다:

- 여행/출장 경비를 카테고리별로 입력하고 싶다
- 실시간 환율로 원화↔현지통화 변환이 필요하다
- 예산을 설정해서 얼마나 썼는지 보고 싶다
- 영수증 사진도 첨부하고 싶다
- 나중에 엑셀이나 파워포인트로 내보내고 싶다
- 모바일에서도 써야 한다

이 정도 요구사항을 던지니까 AI가 알아서 기술 스택을 정하고, 프로젝트를 세팅하고, 페이지를 하나하나 만들어나갔다.

## 기술 스택: 현역 프론트엔드 엔지니어 선택

AI가 선택한 스택은 이랬다:

- **React 19** + **Vite** + **TypeScript** + **Tailwind CSS 4**
- Excel/PPTX 내보내기: `xlsx`, `pptxgenjs`
- 아이콘: `lucide-react`
- 환율 API: exchangerate-api.com

솔직히 내가 선택했어도 비슷하게 갔을 것이다. React + TypeScript는 이제 웹앱의 기본이고, Vite는 빌드가 빠르고, Tailwind는 스타일링이 편하다. 여기에 Docker + Nginx로 배포하고, Caddy를 리버스 프록시로 앞에 세우는 구성까지 AI가 알아서 해줬다.

## 핵심 기능: 생각보다 많이 만들었다

처음에는 단발성 앱이었다. 하나의 여행에 대한 경비 관리. 그런데 쓰다 보니까 "아, 여러 여행을 동시에 관리하고 싶다"는 생각이 들었다. 그래서 다중 여정 관리로 바꿨다.

### 다중 여정 관리

여행이나 출장을 하나의 "여정(Trip)"으로 관리한다. 각 여정마다 독립적으로 경비를 기록할 수 있다. 도쿄 출장, 방콕 가족여행, 부산 출장 등 여러 여정을 만들어놓고 왔다갔다하면서 경비를 입력할 수 있다.

여행과 출장을 구분할 수 있게 했다. 출장이면 💼 아이콘, 여행이면 🏖️ 아이콘이 붙는다. 별거 아닌 것 같은데, 나중에 리포트를 뽑을 때 구분이 되니까 편하다.

### 22개국 통화 + 실시간 환율

한국(KRW), 미국(USD), 일본(JPY), 유럽(EUR), 영국(GBP), 태국(THB), 베트남(VND) 등 22개국 통화를 지원한다. 국내 출장이나 국내 여행도 있으니까 한국 통화도 당연히 포함했다.

환율은 exchangerate-api.com에서 실시간으로 가져온다. 대시보드에서 언제든 새로고침할 수 있고, 마지막 업데이트 날짜도 보여준다. 경비를 입력할 때 원화로 입력하면 현지 통화로 자동 변환해서 보여주고, 반대로 현지 통화로 입력하면 원화로 변환해준다.

### 예산 관리

예산을 설정할 수 있다. 원화 기준으로 할지, 현지 통화 기준으로 할지 선택 가능. 예산을 설정하면 대시보드에 프로그레스 바가 나타나서 얼마나 썼는지 한눈에 볼 수 있다. 80% 넘어가면 노란색, 100% 넘어가면 빨간색으로 바뀐다.

예산 없는 여정도 만들 수 있다. 예산이 꼭 필요한 건 아니니까.

### 동행자, 메모, 영수증

여정에 동행자를 등록할 수 있다. "김대리", "박과장" 같은 이름을 넣어두면, 경비를 입력할 때마다 누구랑 같이 썼는지 선택할 수 있다. 나중에 동행자별 지출 통계도 볼 수 있다.

각 경비에는 메모를 남길 수 있다. "조식", "점심 - 라멘", "공항→호텔 택시" 같은 식으로. 그리고 영수증 사진을 첨부할 수 있다. 카메라로 바로 찍거나 갤러리에서 선택. 5MB까지 가능하다.

### Excel / PowerPoint 내보내기

경비 내역을 Excel이나 PowerPoint로 내보낼 수 있다. 엑셀에는 날짜, 카테고리, 설명, 동행자, 금액, 환율이 표 형식으로 정리된다. 파워포인트에는 요약 슬라이드 + 표 + 영수증 이미지 슬라이드가 포함된다.

출장 보고서를 쓸 때 이게 진짜 편하다. 그냥 내보내기 버튼 누르고 메일로 보내면 끝.

## 모바일 + 배포: 어디서든 접속

Docker로 컨테이너화해서 Mac mini에서 돌리고 있다. Caddy를 리버스 프록시로 세워서 Tailscale 네트워크를 통해 어디서든 접속할 수 있게 했다.

주소는 `블러 처리된 접속 주소`. 아이폰에서 Safari로 열면 모바일 앱처럼 동작한다. 하단에 네비게이션 바가 있어서 대시보드, 경비추가, 내보내기, 여정편집, 여정목록을 오갈 수 있다.

Safari에서 "홈 화면에 추가"를 하면 앱 아이콘처럼 홈 화면에 뜬다. 상태바도 투명하게 처리해서 네이티브 앱처럼 보인다.

## 데이터 동기화: localStorage의 한계를 넘어서

여기서 제일 골치 아팠던 문제가 나왔다.

로컬(Mac mini)에서 데이터를 입력하고, 아이폰에서 접속했는데 데이터가 안 보이는 거다. 당연한 얘기다. localStorage는 브라우저별로 독립적이니까. Mac mini의 Chrome과 아이폰의 Safari는 서로 다른 브라우저다.

그래서 JSON 백업/가져오기 기능을 만들었다. 로컬에서 "백업 파일 저장"을 누르면 모든 데이터(여정, 경비, 영수증)가 JSON 파일로 다운로드된다. 이 파일을 아이폰에서 "백업 파일 가져오기"로 불러오면 된다.

### 영수증 문제: 5MB의 벽

초기에는 백업 파일에 영수증 이미지(base64)를 포함해서 내보냈다. 그런데 아이폰에서 가져올 때 문제가 생겼다. 영수증 사진이 포함된 데이터가 localStorage의 5MB 용량을 초과해버린 거다. Safari의 localStorage 용량이 5MB이기 때문이다.

결국 경비는 가져와졌는데 영수증 이미지는 저장이 안 되는 상황. "껍데기만 가져왔다"는 느낌이었다.

해결책은 **IndexedDB로의 마이그레이션**이었다. localStorage는 5MB가 한계지만, IndexedDB는 수백 MB까지 저장할 수 있다. `src/utils/db.ts`에 IndexedDB 래퍼를 만들고, expenses 저장소를 localStorage에서 IndexedDB로 옮겼다. 여정(trips) 데이터는 작으니까 localStorage에 그대로 두고.

마이그레이션은 자동으로 처리된다. 기존 localStorage에 있던 경비 데이터가 있으면 IndexedDB로 자동 이관한다. 사용자가 느낄 건 없다.

이제 영수증 이미지를 포함한 완전체 백업이 가능하다. 로컬에서 백업 → 아이폰에서 가져오기 → 여정, 경비, 영수증 전부 복원된다.

## v1.0: 버전 마킹과 아이콘

v1.0에서는 앱을 "제품"다운 모습으로 다듬었다.

버전 상수를 추가하고(`APP_VERSION = '1.0.0'`), 화면 하단에 "Money Control v1.0.0 © 2026 All Rights Reserved" 저작권 표시를 넣었다.

파비콘도 만들었다. 보라색 배경에 흰색 ₩ 기호, 금화 포인트가 있는 SVG 아이콘. 아이폰 홈 화면에 추가할 때 쓰는 Apple Touch Icon(180×180 PNG)도 Python Pillow로 생성했다. 아이폰에서 홈 화면에 추가하면 이 보라색 ₩ 아이콘이 앱처럼 뜬다.

PWA 메타태그도 추가했다. `apple-mobile-web-app-capable`로 풀스크린 모드 지원, `apple-mobile-web-app-status-bar-style: black-translucent`로 상태바 투명 처리, `theme-color: #7C3AED`로 브라우저 UI 색상 통일.

## v1.1: 출장 실무를 위한 업그레이드

v1.0을 한동안 쓰다 보니 몇 가지가 더 필요해졌다.

### 카테고리 추가

기존 12개 카테고리에 4개를 추가했다: 로밍/통신비 📶, 렌트/대여 🚗, 회식/접대 🍻, 교육/세미나 📚. 출장 가면 로밍비, 렌터카, 회식비가 빠지면 안 되니까.

### 예산 연동 토글

이게 v1.1의 핵심 기능이다.

출장 가면 "경비성 통제" 항목과 "비통제" 항목이 있다. 예를 들어 항공비와 숙박비는 회사 예산에서 나니까 예산에 포함해야 하지만, 거래처와의 회식비는 별도 정산이니까 예산에서 제외하고 싶다.

그래서 경비를 입력할 때 "예산에 포함 / 예산에서 제외"를 선택할 수 있게 했다. 여정에 예산이 설정되어 있을 때만 이 토글이 보인다.

대시보드에서는 세 가지 숫자를 보여준다:
- **예산 반영 지출**: 예산에 포함된 항목만 합산
- **예산 제외**: 제외된 항목의 건수와 금액
- **총 지출**: 전체 합계

엑셀과 파워포인트에도 "예산 연동" 컬럼이 추가됐다. "포함"인지 "제외"인지 표시되고, 합계도 "전체 합계"와 "예산 반영 합계"로 분리해서 보여준다.

### 경비 편집 기능

v1.0까지는 경비를 추가하거나 삭제만 할 수 있었다. 잘못 입력하면 삭제하고 다시 입력해야 했다. v1.1에서는 편집 기능을 추가했다.

대시보드의 최근 경비 목록에서 각 항목에 ✏️ 편집 버튼이 생겼다. 누르면 기존 데이터가 그대로 채워진 편집 폼으로 이동한다. 날짜, 카테고리, 금액, 메모, 영수증, 예산 연동 여부까지 전부 수정 가능하다. 수정 완료하면 대시보드로 돌아온다.

AddExpensePage를 추가/편집 겸용으로 리팩토링했다. `editingExpense` prop의 유무로 모드를 전환한다. 편집 모드에서는 헤더가 "경비 수정", 버튼이 "경비 수정"으로 바뀐다.

이 과정에서 재밌는 버그도 있었다. 편집 버튼을 눌러도 페이지가 안 바뀌는 현상. 원인은 `ExpenseEditRoute` 컴포넌트가 `useAppData`를 새로 호출하면서, IndexedDB에서 아직 데이터를 로드하기 전에 빈 배열로 경비를 찾으려고 했기 때문이었다. 해결은 간단했다. 상위 컴포넌트에서 이미 로드된 `activeExpenses`를 prop으로 직접 전달하고, `dbReady` 상태도 함께 전달해서 IndexedDB 로딩이 완료될 때까지 렌더링을 보류하는 것.

## iOS Safe Area: 상태바와의 충돌

아이폰에서 쓰다가 화면 상단이 상태바(시간, 와이파이, 배터리)와 겹치는 문제를 발견했다. 특히 Dynamic Island가 있는 모델에서는 더 심했다.

CSS `env(safe-area-inset-*)` 값을 사용해 해결했다:
- `#root`에 `padding-top: env(safe-area-inset-top)` 적용 → 상태바 영역 확보
- 하단 네비게이션에 `paddingBottom: env(safe-area-inset-bottom)` → 홈 인디케이터 영역 확보
- 좌우에도 `env(safe-area-inset-left/right)` 적용 → 가로모드 대응

`viewport-fit=cover` 메타태그가 이미 설정되어 있어서 `env()` 값이 정상 작동했다. 이 한 줄 때문에 화면이 깔끔해졌다.

## AI 에이전트와의 협업: 솔직한 평가

Money Control을 만들면서 AI 에이전트(Sisyphus/OpenCode)와 함께 작업했다. 솔직한 소감을 적어보면:

**잘한 점:**
- 초기 프로젝트 세팅부터 배포까지 한 번에 처리
- 문제가 생기면 원인을 빠르게 분석 (localStorage 용량 문제, IndexedDB 타이밍 이슈)
- Playwright로 브라우저에서 직접 테스트하면서 디버깅
- 변경사항이 많아도 파일 간 의존성을 잘 추적해서 일관성 유지

**아쉬운 점:**
- 초반에 요구사항을 내가 명확히 안 해서 단발성으로 만들었다가 다중 여정으로 리팩토링
- "모바일에서 안 된다"고만 하면 원인 파악에 시간이 걸림. 구체적으로 어떤 문제인지 같이 디버깅해야 빠름

전체적으로, 기능 하나 추가할 때마다 "이것도 필요할 것 같은데?"라고 먼저 물어보거나, 문제를 발견하면 근본 원인까지 파악해서 수정하는 게 인상적이었다. 단순한 코드 생성기가 아니라 실제 개발자와 페어 프로그래밍하는 느낌이었다.

## 최종 구성

Money Control v1.1의 파일 구성:

```txt
money-control/
├── src/
```
