# 콘텐츠 작성 방식

블로그가 어느 정도 구조를 갖추자 다음 문제가 등장했습니다.

글을 어떻게 작성할 것인가?

Next.js 블로그에서는 보통 다음 방식으로 글을 작성합니다.

`posts` 폴더 안에 MDX 파일을 생성합니다.

예를 들어

`posts/my-post.mdx`

이 파일 하나가 블로그 글 하나가 됩니다.

---

# MDX 방식의 장점

MDX는 Markdown과 React를 결합한 형식입니다.

이 방식에는 여러 장점이 있습니다.

- Markdown으로 글 작성 가능
- React 컴포넌트 사용 가능
- Git 기반 콘텐츠 관리 가능

즉 콘텐츠와 코드가 같은 저장소에서 관리됩니다.

---

# 처음 사용한 글 작성 방식

처음에는 글을 다음 방식으로 작성했습니다.

ChatGPT에서 글 생성  
→ 텔레그램으로 전달  
→ OpenClaw가 파일 생성

이 방식은 처음에는 매우 편리했습니다.

AI가 글을 만들고  
OpenClaw가 파일을 생성하면  
블로그에 글이 올라갔습니다.

하지만 곧 큰 문제가 발생했습니다.

---

# 텔레그램 글 잘림 사건

어느 날 블로그 글을 확인했는데 이상한 문제가 발견되었습니다.

글이 중간에서 끊어져 있었습니다.

예를 들어 다음과 같은 형태였습니다.

본문 간략화

처음에는 이유를 이해할 수 없었습니다.

글을 분명히 길게 작성했는데  
블로그에는 일부만 표시되었습니다.

---

# 원인을 찾기 시작하다

문제를 해결하기 위해 다음 과정을 반복했습니다.

- 글 다시 생성
- 파일 확인
- Git 커밋 확인
- 블로그 렌더링 확인

하지만 결과는 동일했습니다.

글이 계속 잘렸습니다.

---

# 원인은 의외의 곳에 있었다

여러 번 테스트한 결과 원인을 찾았습니다.

텔레그램 메시지 길이 제한

텔레그램은 메시지 길이가 길어지면  
내용을 잘라버리는 경우가 있습니다.

즉 문제는 Next.js도 아니었고  
OpenClaw도 아니었습니다.

단순히 **콘텐츠 전달 방식의 문제**였습니다.

---

# 해결 방법

문제 해결 방법은 의외로 간단했습니다.

글을 메시지로 전달하지 않고  
**파일로 직접 관리하는 것**이었습니다.

예를 들어

`posts/post-name.mdx`

파일을 직접 생성하고  
내용을 붙여 넣는 방식입니다.

---

# Git 기반 콘텐츠 관리

이 방식은 다음과 같은 장점이 있습니다.

- 글이 잘릴 가능성 없음
- 버전 관리 가능
- 수정 기록 확인 가능

결국 블로그 콘텐츠는 다음 구조로 관리하게 되었습니다.

글 작성  
→ MDX 파일 생성  
→ Git commit  
→ Git push  
→ Vercel 자동 배포

이 방식이 가장 안정적이었습니다.

---

# 두 번째 문제: 카테고리 필터

콘텐츠 관리가 안정되자 또 다른 문제가 등장했습니다.

카테고리 기능이 제대로 작동하지 않았습니다.

예를 들어 다음과 같은 상황이 발생했습니다.

카테고리 버튼을 클릭하면

`/posts?category=Foodie`

URL은 바뀌지만  
포스트 목록은 그대로였습니다.

즉 필터가 작동하지 않았습니다.

---

# 문제 재현

문제를 재현하기 위해 여러 테스트를 진행했습니다.

예를 들어 다음과 같은 카테고리를 만들었습니다.

- Blog
- Foodie
- A.I
- Life

그리고 포스트에 카테고리를 설정했습니다.

하지만 결과는 동일했습니다.

카테고리를 클릭해도  
모든 포스트가 계속 표시되었습니다.

---

# 코드 분석

문제를 분석하기 위해 posts 페이지 코드를 확인했습니다.

핵심 코드는 다음과 같은 구조였습니다.

전체 포스트 목록을 가져온 후  
카테고리에 따라 필터링하는 방식입니다.

하지만 실제 코드를 확인해 보니 문제가 있었습니다.

필터 로직이 제대로 작동하지 않았습니다.

---

# 필터 로직 수정

문제를 해결하기 위해 필터 코드를 수정했습니다.

기존 로직은 모든 포스트를 그대로 출력하고 있었습니다.

그래서 다음과 같은 조건을 추가했습니다.

카테고리가 All이 아닌 경우  
해당 카테고리 포스트만 표시

이 수정으로 필터 기능이 정상 작동하기 시작했습니다.

---

# 하지만 문제는 여기서 끝나지 않았다

카테고리 필터는 로컬에서는 정상 작동했습니다.

하지만 배포된 사이트에서는 여전히 문제가 있었습니다.

카테고리를 클릭하면

URL은 변경되지만  
포스트 목록은 바뀌지 않았습니다.

---

# Next.js 캐싱 문제

원인을 분석해 보니 Next.js의 캐싱 때문이었습니다.

Next.js는 기본적으로 페이지를 정적으로 생성합니다.

그래서 다음과 같은 문제가 발생했습니다.

URL이 변경되어도  
페이지가 캐시된 상태로 유지되었습니다.

즉 필터가 적용되지 않았습니다.

---

# 해결 방법

이 문제를 해결하기 위해 페이지를 동적으로 렌더링하도록 설정했습니다.

Next.js에서는 다음 설정을 사용할 수 있습니다.

페이지를 동적 렌더링 모드로 변경

이 설정을 적용하자  
카테고리 필터가 정상적으로 작동했습니다.

# AI 개발의 현실

AI 도구는 매우 강력합니다.

하지만 AI가 모든 문제를 해결해 주는 것은 아닙니다.

특히 다음과 같은 문제는 사람이 직접 분석해야 합니다.

- 환경 문제
- 캐싱 문제
- 배포 문제

AI는 해결을 도와줄 수 있지만  
문제를 발견하는 과정은 여전히 사람의 역할입니다.
