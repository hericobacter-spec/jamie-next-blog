# Zhipu AI GLM 모델 가이드: GLM-5부터 경량 모델까지, 그리고 OpenClaw 최적화 활용법

2026년 2월 11일, Zhipu AI(Z.ai)가 **GLM-5**를 출시했다. 744B 파라미터의 MoE(Mixture of Experts) 오픈소스 모델로, DeepSeek V3.2, Kimi K2.5와 함께 **중국 오픈소스 LLM 빅3**를 구성한다. 특히 OpenClaw와 Claude Code와의 공식 호환성을 갖춘 유일한 중국 모델로, 글로벌 AI 생태계에서 독특한 위치를 차지하고 있다.

이 글에서는 GLM 모델 패밀리 전체를 개요부터 가격, 벤치마크, OpenClaw 연동, 활용법까지 상세히 정리한다.

---

## Zhipu AI(Z.ai) 개요

**Zhipu AI(智谱AI)**는 2019년 칭화대학교 KEG 연구실에서 분사한 중국의 AI 기업이다. GLM(General Language Model) 시리즈를 개발하며, 2026년 현재 **세계 Top 3 오픈 가중치 플랫폼** 중 하나로 평가받는다.

### 핵심 특징

- **오픈소스 전략**: GLM-5를 GitHub와 Hugging Face에 공개 (Apache 2.0 라이선스)
- **OpenAI API 호환**: `/v4/chat/completions` 엔드포인트로 OpenAI SDK 그대로 사용 가능
- **Claude Code 호환**: 공식적으로 Claude Code 모델 교체 지원
- **OpenClaw 네이티브 지원**: OpenClaw의 Z.AI 프로바이더로 기본 통합
- **한국어 지원**: GLM-4 시리즈부터 한국어 학습 데이터 포함, GLM-5에서 향상

### 아키텍처 핵심: DeepSeek Sparse Attention (DSA)

GLM-5는 DeepSeek가 개발한 **DSA(DeepSeek Sparse Attention)**를 통합했다. 이는 긴 컨텍스트 처리 비용을 크게 줄이면서도 200K 컨텍스트 윈도우를 유지하는 혁신적인 어텐션 메커니즘이다. 결과적으로 744B 파라미터에도 불구하고 배포 비용을 획기적으로 절감했다.

### 분석

**GLM-5가 특히 강한 분야:**
- **BrowseComp (웹 브라우징)**: 62.0%로 Claude Opus 4.5(37.0%), Gemini 3 Pro(37.8%)를 압도

## OpenClaw 연동 설정

OpenClaw는 Z.AI를 **네이티브 프로바이더**로 지원한다. 설정은 매우 간단하다.

### 1. Z.AI API 키 발급

1. [Z.AI Open Platform](https://z.ai/model-api)에 가입
2. [API Keys](https://z.ai/manage-apikey/apikey-list)에서 API 키 생성
3. [GLM Coding Plan](https://z.ai/model-api/glm-coding-plan) 구독

### 2. OpenClaw 설정

설치 시 Z.AI 프로바이더를 선택하면 자동으로 설정된다:

```bash
openclaw onboard --install-daemon
# Model/auth provider → Z.AI 선택
```

또는 기존 설정에서 모델을 변경:

```json
{
  "primary": "zai/glm-5.1",
  "fallbacks": ["zai/glm-4.7"]
}
```

### 3. 모델 선택 가이드

| 용도 | 추천 모델 | 이유 |
|------|---------|------|
| **메인 에이전트 (품질 우선)** | `zai/glm-5.1` | 최신 프론티어, 에이전트 작업 최적화 |
| **메인 에이전트 (비용 절감)** | `zai/glm-4.7` | 40% 저렴, 대부분 작업 충분 |
| **빠른 응답 (채팅, 요약)** | `zai/glm-4.7-flashx` | 1/14 가격, 단순 작업에 최적 |
| **이미지 분석** | `zai/glm-4.6v` | 네이티브 비전 지원 |
| **초저비용 백그라운드** | `zai/glm-4.5-air` | 허트비트 등 가벼운 정기 작업 |

### 4. 페일오버 설정

```json
{
  "primary": "zai/glm-5.1",
  "fallbacks": ["zai/glm-4.7", "zai/glm-4.6", "zai/glm-4.5-air"]
}
```

고부하 시 자동으로 하위 모델로 전환되어 안정성을 확보한다.

### 5. Claude Code 호환

GLM-5는 Claude Code의 백엔드 모델로도 사용 가능하다:

```json
// ~/.claude/settings.json
{
  "model": "GLM-5"
}
```

Z.ai 공식 문서에서 Claude Code 호환성을 명시적으로 지원한다.

## 실제 활용법

### 에이전트 엔지니어링

GLM-5의 가장 큰 강점은 **장기 수평(long-horizon) 에이전트 작업**이다. Vending Bench 2(1년 기간 자판기 사업 시뮬레이션)에서 $4,432의 최종 잔액을 기록하며, Claude Opus 4.5($4,967)에 근접한 성능을 보여주었다.

### 코드 생성 및 SWE 작업

SWE-bench Verified에서 77.8%, SWE-bench Multilingual에서 73.3%를 기록. 특히 다국어 소프트웨어 엔지니어링에서 GPT-5.2(72.0%)와 Gemini 3 Pro(65.0%)를 상회한다.

### 웹 브라우징 및 검색

BrowseComp에서 62.0%로 압도적 성능. 이는 OpenClaw의 웹검색, 웹페치 기능과 결합할 때 특히 강력하다.

### MCP(Model Context Protocol) 도구 사용

MCP-Atlas에서 67.8%를 기록하며, OpenClaw의 스킬 시스템(웹검색, 캘린더, 파일 관리 등)과 자연스럽게 연동된다.

### Thinking Mode

GLM-5는 기본적으로 Thinking Mode가 활성화되어 있다. 복잡한 추론 문제에서 내부적으로 단계별 사고 과정을 거친 후 답변을 생성한다. `reasoning` 파라미터로 제어 가능하다.

## 로컬 배포

GLM-5는 완전한 오픈소스이므로 로컬에서도 배포 가능하다:

```bash
# vLLM
vllm serve zai-org/GLM-5-FP8 \
  --served-model-name glm-5-fp8

# SGLang (Hopper GPU)
docker pull lmsysorg/sglang:glm5-hopper

# SGLang (Blackwell GPU)
docker pull lmsysorg/sglang:glm5-blackwell

# KTransformers, xLLM도 지원
```

단, 744B 파라미터이므로 최소 4×A100 80GB 또는 동급 GPU가 필요하다. FP8 양자화 버전으로 메모리 요구량을 줄일 수 있다.
