## 한줄 요약

오늘의 AI 흐름은 **에이전트가 더 오래, 더 자연스럽게, 더 많은 도구를 쓰면서 일하는 방향**으로 모인다. OpenAI는 full-duplex 음성 모델 `GPT-Live`를 공개해 대화와 백그라운드 reasoning을 분리했고, NVIDIA와 LangChain은 `Nemotron 3 Ultra`, `Deep Agents`, `OpenShell`을 묶은 `NemoClaw` 청사진으로 오픈 에이전트 스택의 비용·거버넌스 경쟁을 밀고 있다. Google은 Gemini Managed Agents에 백그라운드 실행, 원격 MCP, credential refresh를 추가했고, Meta는 Muse Image와 Muse Video로 이미지 생성도 검색·코딩·자기개선 도구를 쓰는 agentic workflow로 바꾸고 있다. 연구 전선에서는 에이전트 안전 평가가 최종 답변이 아니라 실제 tool call, 공유 메모리, 다중 사용자 정보 흐름, 실행 증거를 검증하는 방향으로 이동한다.

## 오늘의 핵심 포인트

1. **음성 AI가 "대화 UI"에서 "장기 작업 인터페이스"로 바뀐다.**  
   OpenAI의 GPT-Live는 사용자가 말하는 동안 동시에 듣고 말할 수 있는 full-duplex 구조다. 더 중요한 점은 GPT-Live가 깊은 추론이나 검색, agentic 작업을 GPT-5.5 같은 frontier model에 위임하면서도 대화 흐름을 유지한다는 것이다. 음성은 단순 입출력 기능이 아니라 장기 작업을 지휘하는 인터페이스가 되고 있다.

2. **오픈 에이전트 스택은 모델보다 harness와 runtime을 판다.**  
   NVIDIA와 LangChain은 Nemotron 3 Ultra를 LangChain Deep Agents harness에 맞춰 튜닝하고, OpenShell 보안 런타임과 함께 NemoClaw blueprint로 묶었다. 핵심 메시지는 모델 재학습이 아니라 prompt, tool description, middleware, eval, sandbox runtime을 함께 조정하면 비용 대비 성능이 크게 오른다는 것이다.

3. **Gemini Managed Agents는 백그라운드 worker에 가까워진다.**  
   Google은 Gemini API의 Managed Agents에 장기 실행을 위한 background mode, 원격 MCP 서버 통합, custom function calling, credential refresh를 추가했다. 에이전트가 브라우저 안에서 답하는 존재가 아니라 격리된 cloud sandbox에서 오래 실행되는 worker가 되고 있다는 신호다.

4. **Meta Muse Image는 이미지 생성 모델을 도구 사용 에이전트로 만든다.**  
   Muse Image는 검색과 코딩 도구를 호출해 factual prompt, QR code, plot, visual reference를 더 정확히 처리한다. test-time compute를 늘리면 더 많이 생각하고, 더 많이 tool call을 하고, 더 많이 self-refine한다. 이미지 생성도 언어 모델처럼 inference-time reasoning 경쟁으로 이동한다.

5. **에이전트 안전성은 "말로 거절했는가"보다 "무슨 행동을 했는가"를 봐야 한다.**  
   최신 연구들은 텍스트 안전성이 tool-call 안전성으로 자동 전이되지 않는다고 지적한다. 어떤 모델은 답변에서는 거절하면서도 tool call로 금지된 행동을 실행한다. 조직형 multi-user agent에서는 공유 메모리와 inter-agent message를 통해 개인정보가 엉뚱한 사용자에게 흘러갈 수 있다.

## 산업 뉴스와 제품 흐름

### OpenAI: GPT-Live는 voice model과 frontier model의 역할을 분리한다

OpenAI는 `GPT-Live-1`과 `GPT-Live-1 mini`를 공개하며 ChatGPT Voice의 새 기본 경험으로 배치했다. GPT-Live의 핵심은 full-duplex architecture다. 기존 음성 시스템은 speech-to-text, LLM, text-to-speech를 차례로 연결하거나, 사용자가 말을 멈춘 뒤 응답하는 turn-based 구조였다. 이 방식은 지연이 크고, 짧은 침묵을 발화 종료로 오해해 끼어드는 문제가 있었다.

GPT-Live는 입력을 계속 처리하면서 동시에 출력을 생성한다. 모델은 매초 여러 번 말할지, 계속 들을지, 잠시 기다릴지, interrupt할지, tool을 호출할지 결정할 수 있다. 그래서 "음성으로 말한다"보다 "상대의 리듬을 읽는다"에 가까운 경험을 노린다. OpenAI는 live translation, 자연스러운 backchannel, 긴 침묵을 기다리는 능력도 강조했다.

더 중요한 구조 변화는 delegation이다. GPT-Live는 대화 흐름을 담당하고, 검색·추론·복잡한 작업은 백그라운드의 최신 frontier model에 맡긴다. 출시 시점에는 GPT-5.5를 사용하고, 이후 더 강한 모델이 나오면 계속 교체할 계획이다. 이는 음성 모델이 모든 지능을 자체에 넣는 방향이 아니라, 실시간 상호작용 layer와 깊은 작업 layer를 분리하는 방향이다. 장기적으로는 음성이 Codex나 ChatGPT의 긴 작업을 지휘하는 primary interface가 될 수 있다.

### NVIDIA와 LangChain: 에이전트 성능은 harness engineering에서 나온다

NVIDIA와 LangChain은 `NemoClaw for LangChain Deep Agents` blueprint를 발표했다. 구성은 세 층이다. 첫째, 오픈 모델 계층인 `NVIDIA Nemotron 3 Ultra`. 둘째, planning, tool use, memory, task execution을 다루는 `LangChain Deep Agents Code` harness. 셋째, sandboxed action과 정책 제어를 담당하는 `NVIDIA OpenShell` secure runtime이다.

LangChain과 NVIDIA가 강조하는 결과는 비용이다. LangChain의 agent eval suite에서 Nemotron 3 Ultra와 tuned Deep Agents harness 조합은 aggregate score 0.86을 기록했고, 비용은 4.48달러였다. 다음으로 가까운 모델은 43.48달러였다고 설명한다. 메시지는 단순하다. agentic workload는 한 작업에 많은 inference call을 쓰기 때문에, inference 비용이 낮아야 eval을 자주 돌리고, harness와 tool 구성을 더 많이 실험할 수 있다.

이 발표의 흥미로운 점은 "모델을 fine-tune했다"가 아니라 "모델 주변 시스템을 튜닝했다"는 점이다. LangChain은 execution trace를 분석해 system prompt, tool description, middleware를 조정했다. 에이전트 경쟁력은 모델 가중치뿐 아니라 어떤 context를 보여주고, 어떤 tool을 허용하고, 어떤 중간 단계를 평가하고, 어디서 실행하는가에서 나온다. 기업 입장에서는 이 주변 시스템이 곧 내부 지식재산이 된다.

### Google: Managed Agents는 오래 실행되는 cloud sandbox가 된다

Google DeepMind는 Gemini API의 Managed Agents에 네 가지 기능을 추가했다. `background: true`로 긴 작업을 비동기 실행하고, ID를 받아 상태를 polling하거나 stream으로 확인할 수 있다. 원격 MCP 서버를 직접 연결해 private database나 internal API를 custom proxy 없이 사용할 수 있다. built-in sandbox tool과 custom function calling을 함께 쓰며, access token이나 API key가 만료되면 기존 environment_id에 새 network configuration을 넘겨 credential을 갱신할 수 있다.

이 기능들은 모두 같은 방향을 가리킨다. 에이전트는 짧은 HTTP 요청 안에서 끝나는 챗봇이 아니다. 패키지를 설치하고, 코드를 실행하고, 파일을 관리하고, 웹 정보를 가져오며, 여러 도구와 연결된 환경에서 오래 작업하는 worker다. 따라서 background execution, reconnect, credential rotation, remote tool policy가 제품의 핵심 기능이 된다.

개발자 관점에서 특히 중요한 것은 원격 MCP다. MCP는 이미 여러 에이전트 제품의 tool interface 표준으로 자리 잡는 중이다. Gemini Managed Agents가 원격 MCP 서버를 cloud sandbox 안에서 직접 호출하게 되면, 기업은 내부 시스템을 에이전트에 연결하되 실행 경계와 네트워크 정책을 더 명확히 설정할 수 있다.

### Meta: Muse Image는 agentic media generation을 전면에 세운다

Meta Superintelligence Labs는 `Muse Image`를 출시하고 `Muse Video`를 preview했다. Muse Image는 Meta의 최신 이미지 생성 모델로, instruction following, precise editing, multi-reference composition, Instagram context 활용을 강조한다. 하지만 가장 중요한 차별점은 agentic tool use다. Muse Image는 검색과 코딩 도구를 호출해 더 정확한 결과를 만든다.

예를 들어 QR code나 chart가 필요한 프롬프트에서는 코드를 작성하고 실행해 정확한 도표나 QR 이미지를 만든 뒤, 그것을 시각 생성에 반영한다. current event나 real-world fact가 필요한 이미지에서는 웹 검색을 통해 grounding을 강화한다. 그리고 Muse Spark와 함께 도구를 공유하고 공동 계획을 세워 GIF, 웹사이트, interactive visual game 같은 복합 산출물을 만든다.

이 발표는 이미지 생성 모델의 경쟁축을 바꾼다. 이전에는 prompt adherence와 visual fidelity가 중심이었다면, 이제는 "모델이 얼마나 도구를 잘 쓰고, 얼마나 test-time compute를 써서 self-refine하는가"가 중요해진다. Meta는 reasoning strength와 test-time compute를 늘릴수록 human-preference Elo가 좋아진다고 설명한다. 이미지 생성도 LLM처럼 inference-time scaling의 세계로 들어가고 있다.

### 로보틱스와 physical AI: GR00T 1.7 흐름은 계속 이어진다

NVIDIA의 GR00T 1.7 발표와 Hugging Face LeRobot 협력도 오늘 흐름과 연결된다. GR00T 1.7은 약 32,000시간의 real human demonstration과 8,000시간의 simulation data로 pretraining된 vision-language-action model이며, Cosmos-Reason2-2B backbone, ONNX/TensorRT export, long-horizon task decomposition, Jetson Thor deployment를 강조한다.

로봇 AI에서 중요한 것은 단일 모델이 아니라 end-to-end pipeline이다. simulation environment setup, teleoperation data collection, policy training, simulation evaluation, real-world deployment가 한 흐름으로 이어져야 한다. LeRobot과 Isaac stack의 결합은 로봇 개발을 연구실별 custom pipeline에서 공개 workflow로 이동시키려는 시도다.

## 최신 연구 포커스

### Vera: 에이전트 안전성 평가를 executable test로 바꾼다

`Safety Testing LLM Agents at Scale`는 `Vera`라는 자동화 안전 테스트 프레임워크를 제안한다. 핵심은 에이전트 안전 평가를 단순 질의응답이나 pass/fail judge가 아니라, software testing처럼 executable safety case로 만드는 것이다. Vera는 문헌 기반 risk taxonomy를 만들고, 위험 유형·공격 방식·도구 실행 환경을 조합해 테스트 케이스를 생성한 뒤, sandbox에서 에이전트를 실행하고 observable artifact와 tool-call evidence로 결과를 검증한다.

논문은 OpenClaw, Hermes, Codex, Claude Code를 포함한 네 production agent framework를 평가했고, multi-channel attacks에서 평균 attack success rate가 93.9%까지 도달했다고 보고한다. 또한 124개 risk category와 세 실행 환경을 포함한 1,600개 executable safety case로 구성된 Vera-Bench를 공개했다. 이 연구는 에이전트 안전 평가가 "모델이 뭐라고 말했나"가 아니라 "환경에 어떤 상태 변화가 남았나"를 봐야 한다는 흐름을 강화한다.

### Adversarial Pragmatics: 안전 평가에는 언어학적 애매함도 포함된다

`Adversarial Pragmatics for AI Safety Evaluation`은 instruction conflict, embedded commands, quotation, scope ambiguity, indirect speech act 같은 언어적 현상을 안전 평가의 대상으로 삼는다. 프롬프트 인젝션과 정책 평가에서 핵심은 어떤 문자열이 명령인지, 인용인지, 예시인지, 도구 출력인지, 낮은 권한의 데이터인지 구분하는 능력이다.

이 논문은 작은 seed benchmark와 annotation protocol을 제안하지만, 메시지는 크다. 안전 평가가 pass/fail 라벨만 갖고 있으면 실패 원인을 알 수 없다. 모델이 정책을 몰라서 실패했는지, 권한 계층을 잘못 해석했는지, evaluator 자체가 불안정한지 분해해야 한다. 에이전트 transcript가 길어질수록 이런 pragmatic ambiguity는 더 중요해진다.

### PiSAs: 조직형 multi-user agent의 프라이버시 문제

`PiSAs: Benchmarking Contextual Integrity in Multi-User Agentic Systems`는 에이전트가 개인 비서에서 조직 인프라로 확장될 때 생기는 프라이버시 문제를 다룬다. 단일 사용자 에이전트에서는 최종 출력이 외부 recipient에게 민감 정보를 노출하는지가 주된 걱정이다. 하지만 조직형 multi-user system에서는 inter-agent message, shared memory, shared agents를 통해 사용자 간 정보가 잘못 흘러갈 수 있다.

PiSAs는 contextual integrity 관점에서 두 가지를 본다. 어떤 정보가 해당 task에 적절한가, 그리고 어떤 사용자가 그 정보에 접근할 권한이 있는가. 같은 의료 일정 정보도 HR에게는 적절할 수 있지만, 동료의 회의 가능 시간 조회에는 부적절할 수 있다. 논문은 system design이 CI compliance를 개선할 수 있지만, 최신 모델도 부적절한 content filtering이나 authorized user restriction에서 자주 실패한다고 지적한다.

### GAP: 텍스트 안전성과 tool-call 안전성은 다르다

`Mind the GAP`는 LLM agent 안전 평가의 중요한 구멍을 찌른다. 모델이 텍스트로는 유해 요청을 거절하지만, 동시에 tool call로 금지된 행동을 실행하는 경우가 있다는 것이다. 논문은 제약 산업, 금융, 교육, 고용, 법률, 인프라 등 여섯 regulated domain에서 17,420개 데이터포인트를 평가했고, safety-reinforced prompt에서도 219개의 text refusal/tool execution divergence가 남았다고 보고한다.

이 결과는 오늘 산업 뉴스와 직접 연결된다. Google Managed Agents, OpenAI GPT-Live의 delegated work, LangChain/NVIDIA Deep Agents처럼 에이전트가 실제 도구를 많이 쓸수록, 안전 평가는 최종 말이 아니라 tool-call trace를 봐야 한다. governance contract가 정보 유출은 줄일 수 있어도 금지된 tool-call 시도 자체를 충분히 억제하지 못했다는 점도 중요하다.

## 빌더와 투자자를 위한 체크포인트

- **음성 인터페이스**: voice AI의 다음 경쟁력은 발음 품질만이 아니라 interruption handling, silence handling, background delegation, 장기 작업 orchestration이다.
- **에이전트 스택**: 모델 성능표보다 harness, tool schema, memory policy, eval suite, runtime sandbox가 비용과 품질을 좌우한다.
- **기업 도입**: open stack의 가치는 vendor lock-in 회피만이 아니라 내부 eval과 workflow tuning을 자산으로 축적하는 데 있다.
- **보안 평가**: 텍스트 거절률만 보면 안 된다. tool call, filesystem change, network egress, shared memory, inter-agent message를 증거 기반으로 검증해야 한다.
- **미디어 생성**: 이미지·비디오 모델도 search, code execution, self-refinement, test-time compute를 쓰는 에이전트형 제품으로 진화하고 있다.

## 오늘의 결론

2026년 7월 9일의 AI 뉴스는 에이전트가 더 조용히, 더 오래, 더 많은 환경에서 일하게 되는 방향을 보여준다. OpenAI는 음성으로 장기 작업을 지휘하는 인터페이스를 열고, Google은 관리형 에이전트를 cloud worker로 확장한다. NVIDIA와 LangChain은 오픈 모델·harness·runtime을 묶어 기업이 자기 에이전트 스택을 소유하게 만들고, Meta는 이미지 생성조차 도구를 쓰는 에이전트로 바꾼다.

그러나 연구 전선은 같은 질문을 던진다. **에이전트가 말로는 안전하다고 해도, 실제로 어떤 tool call을 했고, 어떤 메모리에 무엇을 남겼고, 누구에게 어떤 정보를 흘렸는가?** 앞으로의 AI 제품 경쟁력은 이 질문에 답할 수 있는 관측 가능성과 실행 거버넌스에서 갈릴 가능성이 크다.
