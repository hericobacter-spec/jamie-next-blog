# 초보가 OpenClaw로 Next.js 블로그 만들기 Part 5

Part 4에서는 다음 내용을 다루었습니다.

- Vercel 배포 시스템
- Preview와 Production의 차이
- Pull Request 기반 개발 방식

이 과정을 거치면서 블로그는 어느 정도 안정적으로 운영되기 시작했습니다.

하지만 블로그를 실제로 운영하려면 또 하나 중요한 문제가 있었습니다.

모바일 포스팅

즉 컴퓨터가 없어도 블로그 글을 작성할 수 있어야 했습니다.

---

# 왜 모바일 포스팅이 필요한가

블로그 글은 항상 컴퓨터 앞에서만 작성되는 것이 아닙니다.

다음과 같은 상황이 자주 발생합니다.

여행 중  
카페에서  
대중교통 안에서

이런 상황에서는 노트북을 열기 어렵습니다.

그래서 스마트폰에서도 글을 작성할 수 있는 방법이 필요했습니다.

---

# 해결책: Working Copy

이 문제를 해결하기 위해 사용한 도구가 바로 **Working Copy**입니다.

Working Copy는 iOS에서 사용할 수 있는 Git 클라이언트입니다.

즉 스마트폰에서도 Git 작업을 할 수 있습니다.

Working Copy를 사용하면 다음 작업이 가능합니다.

- GitHub 저장소 clone
- 파일 편집
- commit
- push

즉 스마트폰에서도 블로그 콘텐츠를 관리할 수 있습니다.

---

# Working Copy 설치

Working Copy는 iOS 앱스토어에서 설치할 수 있습니다.

설치 후 다음 단계를 진행합니다.

GitHub 계정 연결

그리고 블로그 저장소를 clone 합니다.

예를 들어 다음과 같은 저장소입니다.

`jamie-next-blog`

clone이 완료되면 스마트폰에서도 프로젝트 파일을 볼 수 있습니다.

---

# 모바일 포스팅 방식

모바일에서 글을 작성하는 과정은 다음과 같습니다.

Working Copy 실행  
→ `posts` 폴더 이동  
→ 새 MDX 파일 생성

예를 들어 다음 파일을 생성합니다.

`posts/my-new-post.mdx`

그리고 Markdown 형식으로 글을 작성합니다.

---

# MDX 기본 구조

블로그 글은 다음 구조로 작성됩니다.

frontmatter  
본문

예를 들어 다음과 같은 형태입니다.

- `title`
- `date`
- `description`
- `category`
- `tags`

그리고 그 아래에 실제 글 내용이 들어갑니다.

---

# 모바일 글 작성의 장점

Working Copy를 사용하면 다음 장점이 있습니다.

언제 어디서든 글 작성 가능

Git 기반 버전 관리

즉 글이 자동으로 기록됩니다.

그리고 commit 후 push 하면 바로 배포가 진행됩니다.

---

# 자동 배포 시스템

블로그의 자동 배포 구조는 다음과 같습니다.

Working Copy에서 commit

→ GitHub push

→ Vercel 자동 빌드

→ 블로그 업데이트

이 과정은 완전히 자동입니다.

즉 스마트폰에서 글을 올리면  
몇 분 후 블로그에 반영됩니다.

---

# SEO 구조 만들기

블로그 운영에서 매우 중요한 요소가 있습니다.

SEO

검색 엔진 최적화입니다.

SEO가 제대로 되어 있지 않으면  
검색 결과에 노출되지 않습니다.

---

# Next.js 블로그 SEO 구성

Next.js 블로그에서는 다음 요소가 중요합니다.

- meta description
- title 태그
- Open Graph
- RSS
- sitemap

이 요소들이 검색 엔진에 영향을 줍니다.

---

# RSS 피드 생성

RSS는 블로그 구독 시스템입니다.

RSS를 통해 다음과 같은 기능이 가능합니다.

블로그 글 자동 구독

RSS 리더 연동

콘텐츠 배포 확장

Next.js에서는 RSS를 직접 생성할 수 있습니다.

---

# RSS 파일 구조

보통 다음 경로에 생성됩니다.

`/rss.xml`

이 파일은 블로그 글 목록을 XML 형식으로 제공합니다.

RSS 리더는 이 파일을 읽어서  
새 글을 자동으로 가져옵니다.

---

# RSS 자동 생성

Next.js에서는 서버 라우트를 이용해 RSS를 생성할 수 있습니다.

예를 들어 다음 구조입니다.

`app/rss.xml/route.ts`

이 파일이 RSS를 생성합니다.

---

# 자동화의 완성

이제 블로그 시스템은 다음 구조를 갖게 됩니다.

글 작성

→ Git commit

→ GitHub push

→ Vercel 빌드

→ SEO 반영

→ RSS 업데이트

모든 과정이 자동으로 진행됩니다.
