# [NEWS] OpenClaw 2026.4.5 대업데이트: 한글 UI, Dreams 메모리 시스템, 비디오·음악 생성까지

> OpenClaw이 2026년 4월 5일 **2026.4.5** 버전을 배포했다. AI 에이전트 플랫폼으로서 미디어 생성, 자동 메모리 정리, 다국어 UI 등 한 단계 도약하는 업데이트다. 실제 업데이트 후 환경에서 주요 변경점을 확인해보았다.

## 이번 업데이트의 규모

이번 2026.4.5는 단순한 버그 수정이 아니다. **에이전트가 직접 비디오와 음악을 생성**할 수 있게 되었고, AI가 자체적으로 메모리를 정리하는 **Dreams 시스템**이 도입되었으며, Control UI가 **13개 언어**를 지원하게 되었다. 신규 프로바이더만 5개 이상 추가되었고, 프롬프트 캐싱 안정화 등 성능 개선도 상당하다.

---

## 🇰🇷 한글 Control UI

가장 먼저 눈에 띄는 변화다. Control UI가 한국어를 비롯해 **간체/번체 중국어, 포르투갈어, 독일어, 스페인어, 일본어, 프랑스어, 터키어, 인도네시아어, 폴란드어, 우크라이나어**까지 총 13개 언어를 지원한다.

![OpenClaw 한글 Control UI](/images/posts/openclaw-2026-04-update/korean-ui.jpg)

실제로 확인해보니 메뉴, 설정 항목, 상태 표시까지 전반적으로 한글로 표시된다. 영어에 익숙하지 않은 사용자에게는 큰 진입 장벽이 낮아졌다. 별도 언어 설정 없이 시스템 로케일을 기반으로 자동 감지되는 것으로 보인다.

---

## 🧠 Dreams: AI가 스스로 정리하는 메모리 시스템

이번 업데이트의 핵심 기능이다. **Dreams(드리밍)**은 에이전트가 작성한 일일 메모를 자동으로 분석하고, 중요한 내용만 장기 기억으로 승격시키는 시스템이다.

### 3단계 메모리 정리

Dreams는 수면의 3단계를 모방하여 동작한다:

| 단계 | 이름 | 역할 |
|------|------|------|
| Light | 가벼운 스캔 | 최근 일일 메모를 읽고 후보를 선별 |
| Deep | 깊은 분석 | 중요도 가중치를 부여하고 노이즈를 제거 |
| REM | 꿈의 정리 | MEMORY.md에 반영할 내용을 최종 결정 |

![Dreams UI](/images/posts/openclaw-2026-04-update/dreams-ui.jpg)

Control UI의 Dreams 메뉴에서 현재 드리밍 상태를 확인할 수 있다. 실제로 확인해보니 아직 첫 드리밍 사이클이 완료되지 않은 상태에서도 UI가 정상적으로 표시되었다.

채팅에서 `/dreaming` 명령어를 입력하면 현재 설정과 상태를 확인할 수 있다:

![dreaming 명령어 결과](/images/posts/openclaw-2026-04-update/dreaming-command.jpg)

### 설정 방법

`openclaw.json`에서 `enabled: true`로 활성화하고, `frequency`로 실행 주기를 설정할 수 있다. 또한 `recencyHalfLifeDays`와 `maxAgeDays`로 메모리 노화(aging)를 제어하여, 얼마나 오래된 메모까지 분석 대상으로 삼을지 조정 가능하다.

### 드리밍 로그

드리밍 과정에서 생성된 내용은 워크스페이스의 `dreams.md`에 기록된다. 이 파일은 기본 리콜 대상에는 포함되지 않으므로, 명시적으로 읽지 않는 한 자동 참조되지 않는다.

## 🎬 비디오 · 음악 생성 툴 내장

에이전트가 이제 텍스트만 출력하는 것이 아니라 **미디어를 직접 생성**할 수 있게 되었다.

### video_generate

내장 `video_generate` 툴을 통해 에이전트가 비디오를 생성할 수 있다. 지원 프로바이더:

- **xAI (grok-imagine-video)**
- **Alibaba Model Studio Wan**
- **Runway**

### music_generate

내장 `music_generate` 툴도 추가되었다. 지원 프로바이더:

- **Google Lyria**
- **MiniMax**
- **ComfyUI** (워크플로우 기반)

비동기 작업 추적(async task tracking)을 지원하므로, 음악 생성이 완료되면 자동으로 결과를 전달한다. 대화 중에 "이 분위기에 맞는 BGM 만들어줘" 같은 요청이 가능해진 셈이다.

## 🔌 새로운 프로바이더 5개 추가

이번 업데이트에서 가장 많은 변화가 있었던 부분 중 하나다.

| 프로바이더 | 유형 | 특징 |
|-----------|------|------|
| **Qwen** | 챗/임베딩 | 알리바바 클라우드 Qwen 모델 시리즈 |
| **Fireworks AI** | 챗 | 빠른 추론 속도 |
| **StepFun** | 챗 | 중국 AI 스타트업 |
| **Amazon Bedrock Mantle** | 다중 모델 | Claude, GPT-OSS, Qwen, Kimi, GLM 등 |
| **MiniMax** | TTS/검색 | 음성 합성 + 웹 검색 |

특히 **Bedrock Mantle**은 AWS IAM 인증 체인을 자동 감지하므로, 수동으로 `AWS_BEARER_TOKEN_BEDROCK`을 설정할 필요가 없다. Bedrock에서 호스팅되는 다양한 모델을 라우팅 없이 직접 사용할 수 있게 되었다.

또한 `openai-codex/gpt-5.4-mini`가 새로 추가되었다. 가벼운 작업에 적합한 비용 효율적 모델이다.

## 🛠 Skills 패널 개선: ClawHub 직접 검색

![Skills / ClawHub 패널](/images/posts/openclaw-2026-04-update/skills-clawhub.jpg)

Control UI의 Skills 패널에서 **ClawHub 검색, 상세 확인, 설치**를 직접 할 수 있게 되었다. 기존에는 터미널에서 `openclaw plugins install`을 실행해야 했지만, 이제 웹 UI에서 브라우징하고 원클릭 설치가 가능하다.

`openclaw plugins install --force` 옵션도 추가되어, 기존에 설치된 플러그인을 강제 교체할 수 있다.

## 🧠 Thinking Level 피커

![Thinking Level 피커](/images/posts/openclaw-2026-04-update/thinking-picker.jpg)

채팅 헤더에 **Thinking 레벨 선택 드롭다운**이 추가되었다. Off, Low, Medium, High, Stream 등 다양한 수준을 세션별로 즉시 전환할 수 있다. 모바일 채팅 설정에서도 동일하게 지원된다.

이전에는 `/thinking` 명령어를 입력해야 했지만, UI에서 직접 조작 가능해져서 워크플로우가 훨씬 매끄러워졌다.

## 🔗 Claude CLI MCP 브릿지

OpenClaw의 툴을 **Claude CLI에 MCP 브릿지**로 노출하는 기능이다. 백그라운드 Claude CLI 실행 중에도 OpenClaw의 툴(파일 읽기, 웹 검색, 캘린더 등)을 Claude가 직접 사용할 수 있다.

또한 `stdin + stream-json` 방식으로 전환되어, 긴 응답도 실시간 프로그레스를 보여주고 최종 메타데이터도 정상적으로 수신된다. Claude Code 사용자에게는 특히 유용한 업데이트다.

---

## 📱 iOS Exec 승인 알림

iOS 사용자를 위한 기능이다. 이제 **APNs 푸시 알림**로 exec 명령 승인 요청이 오고, 인앱 모달에서 직접 승인/거부할 수 있다. 이전에는 Telegram이나 다른 채널을 통해 승인해야 했지만, iPhone에서 모든 것이 해결된다.

---

## 🚀 프롬프트 캐싱 안정화

기술적으로 중요한 변경이다. 프롬프트 접두사 재사용성, 결정론적 MCP 툴 정렬, 컴팩션, 임베디드 이미지 히스토리 등 다양한 경로에서 캐시 적중률이 개선되었다.

`openclaw status --verbose`에서 캐시 진단 정보를 확인할 수 있으며, 시맨틱하게 동일한 프롬프트가 KV/캐시를 더 안정적으로 재사용하게 되었다. API 비용 절감에 직결되는 개선이다.

---

## ⚠️ Breaking Changes

기존 설정에서 일부 레거시 별칭이 제거되었다:

- `talk.voiceId` / `talk.apiKey` → 정식 경로로 마이그레이션
- `agents.*.sandbox.perSession` → `enabled`로 변경
- `browser.ssrfPolicy.allowPrivateNetwork` → 정식 경로
- `hooks.internal.handlers` → 새 훅 시스템

`openclaw doctor --fix`로 자동 마이그레이션이 가능하다. 업데이트 후 실행해보는 것을 추천한다.

## 총평

OpenClaw 2026.4.5는 **텍스트 에이전트에서 멀티미디어 에이전트로의 전환**을 공식적으로 선언하는 업데이트다. 비디오와 음악 생성이 내장되고, Dreams 시스템으로 에이전트가 자체적으로 메모리를 관리하며, 13개 언어 UI로 글로벌 접근성이 크게 향상되었다.

한국 사용자 입장에서는 **한글 UI 공식 지원**이 가장 환영할 만한 변화다. Dreams 시스템은 실험적이지만, 에이전트가 장기적으로 유용한 비서가 되기 위한 필수 인프라다. 앞으로의 발전이 기대된다.
