## 1. 🔥 Claude Mythos 5: 최초의 10조 파라미터 모델

**Anthropic**이 **Claude Mythos 5**를 출시했다. 인류 최초로 널리 인정받는 **10조(10 Trillion) 파라미터** 모델이다.

### 핵심 특징

- **전문 특화 설계**: 사이버보안, 학술 연구, 복잡한 코딩 환경에서 기존 소형 모델이 겪던 "청크 스킵핑(chunk-skipping)" 오류를 근본적으로 해결
- **다단계 추론 정밀도**: 방대한 파라미터를 활용해 이전에는 인간 전문가에게만 가능했던 수준의 장거리 계획 수립 정밀도 달성
- **"전문 밀도(specialized density)" 패러다임**: 단순히 파라미터 수를 늘린 것이 아니라, 다단계 추론의 정밀도를 위해 특화된 아키텍처 설계

Mythos 5는 일반적인 대화용 모델이 아니다. 고위험 환경(high-stakes environment)에서 사용되며, Anthropic의 프론티어 모델 라인업에서 특수 목적 최상위 모델의 위치를 차지한다.

## 3. 🚨 Claude Code 소스코드 유출: 50만 줄이 공개

### 사건 개요

- **발생일**: 2026년 3월 31일
- **원인**: Claude Code npm 패키지 업데이트(v2.1.88)에 **디버그용 소스맵(.map) 파일**이 실수로 포함
- **규모**: **50만 줄 이상**의 소스코드 공개
- **발견자**: 보안 연구원 Chaofan Shou

### 원인 규명

Boris Cherny (Claude Code 제작자)가 직접 원인을 확인:

> "자동화되어야 할 수동 배포 단계가 있었다. **절대 개인의 잘못이 아니라 시스템의 문제**다."

초기에는 Bun 런타임의 버그로 추정되었으나, Cherny가 직접 **개발자 오류(developer error)**임을 인정했다.

### 유출로 밝혀진 주요 내용

1. **"Undercover Mode"**: Anthropic 직원이 공개 레포지토리에 커밋할 때 AI 저작자 표기와 Anthropic 코드네임을 자동으로 제거하는 기능
2. **오케스트레이션 레이어**: 실제 AI 모델 관련 코드는 전체의 **1.6%**에 불과 — 나머지는 도구 조율, 프롬프트 캐싱, 에이전트 워크플로우 로직
3. **MAGIC DOC 헤더**: Anthropic 내부 파일 시스템의 특수 문서 포맷

이 사건은 **AI 코딩 도구의 핵심 가치가 모델 자체가 아니라 오케스트레이션 레이어**에 있음을 명확히 보여주었다. 소스코드가 공개되었음에도 Claude Code의 가치는 여전히 **$25억 연간 매출**을 기록 중이다.

## 4. 🏢 Microsoft의 다중 모델 전환: OpenAI 독점에서 벗어나다

Microsoft가 자사 AI 연구 도구 **Critique**를 리뉴얼하며 **OpenAI와 Anthropic 모델을 모두 사용**하도록 전환했다.

### 의미

- **"멀티 모델(multi-model) 미래"**의 명확한 신호
- 주요 프론티어 랩이 **서로를 번갈아 추월(leapfrog)**하는 상황에서, 기업들은 하나의 모델 공급자에 종속되지 않으려 함
- Microsoft 365 Copilot에도 Anthropic 모델이 도입된 것으로 확인

추가로 Bloomberg 보도에 따르면, Microsoft는 **2027년까지 자체 대형 최첨단 AI 모델**을 개발하는 것을 목표로 하고 있다. Mustafa Suleyman (Microsoft AI CEO)은 "향후 몇 년간 AI 산업 전체가 [특정 방향으로] 정의될 것"이라고 발표했다.

## 5. 🚀 프론티어 모델 성능 현황 (4월 2-3일 기준)

| 모델 | 개발사 | 파라미터/아키텍처 | 주요 용도 | 핵심 성능 |
|------|--------|------------------|----------|----------|
| Claude Mythos 5 | Anthropic | 10조 | 보안·연구 | 다단계 계획 최고 수준 |
| GPT-5.4 Thinking | OpenAI | 독자적 (Post-Training Scaling) | OS 수준 에이전트 | OSWorld 75.0%, GDPVal 83.0% |
| Gemini 3.1 Ultra | Google DeepMind | 네이티브 멀티모달 | 실시간 비전/음성 | GPQA Diamond 94.3% |
| Grok 4.20 | xAI (SpaceX) | 4-에이전트 협업 | 팩트 체크·웹 | 환각률 22% (비환각 78%) |
| DeepSeek V4 | DeepSeek | 1조 (Open MoE) | 코딩·수학 | HumanEval 94.7% |
| Gemma 4 31B | Google (오픈) | 31B Dense | 로컬 에이전트 | Arena AI #3 |

### 주목할 점

- **GPT-5.4 Thinking**은 OSWorld-Verified 벤치마크에서 75.0%를 기록하며 GPT-5.2 대비 **+27.7 포인트** 향상. 이는 **인간 수준의 데스크톱 작업 능력**을 공식적으로 초과한 것.
- **Gemini 3.1 Flash-Lite**는 이전 세대 대비 응답 속도 **2.5배 빠르고**, 출력 생성 속도 **45% 향상**.
- **DeepSeek V4**는 1조 파라미터 오픈 MoE로 HumanEval 94.7% 기록 — 프론티어급 성능을 오픈소스로 제공.

## 6. 📉 Google TurboQuant: AI 칩 시장을 뒤흔든 양자화 혁신

Google이 **TurboQuant**를 발표했다. 프론티어 모델의 성능을 유지하면서 **메모리 요구량을 6분의 1로 줄이는** 양자화(quantization) 알고리즘이다.

### 영향

- AI 칩 업계에 큰 타격 — 더 적은 VRAM으로 더 큰 모델을 구동 가능
- 엣지 디바이스와 소비자 GPU에서 프론티어급 모델 배포의 경제성 획기적 개선
- "원시 스케일링(raw scaling)"과 "효율적 압축(efficient compression)" 간의 긴장이 AI 산업의 핵심 축으로 부상
