## 한줄 요약

오늘의 AI 흐름은 **에이전트가 실험실 데모를 넘어 조직의 실제 업무, 국방, 브라우저 조작, 오픈 모델 생태계, 물리 시스템으로 들어가면서 핵심 경쟁 축이 성능에서 실행 통제로 이동하고 있다**는 말로 정리된다. OpenAI는 Codex 사용 데이터를 통해 지식 노동의 단위가 짧은 질의응답에서 장시간 위임 작업으로 바뀌고 있다고 설명했고, 국방 영역에서는 클라우드 기반 배포, 안전 스택 유지, 국내 감시와 자율무기 금지 같은 레드라인을 전면에 세웠다. Anthropic도 미국 국방부와 2년, 2억 달러 한도의 프로토타입 계약을 발표하면서 책임 있는 배포와 Claude Gov 모델을 강조했다. Google은 Gemini 3.5 Flash에 computer use를 내장해 브라우저, 모바일, 데스크톱 환경을 조작하는 에이전트 개발을 더 직접적으로 지원하기 시작했다. NVIDIA는 Nemotron 3 Ultra를 장시간 에이전트용 오픈 모델로 공개하며, 에이전트 인프라가 open weights, agent traces, secure runtime, speech, safety model까지 묶인 생태계 경쟁으로 커지고 있음을 보여줬다.

- OpenAI와 Anthropic 모두 국방 AI 계약을 공개적으로 설명하며, frontier AI 기업이 국가안보 수요와 안전 레드라인을 동시에 다루는 국면으로 들어갔다.
- Google은 Gemini 3.5 Flash에 computer use를 내장하고, 민감하거나 되돌릴 수 없는 행동에는 사용자 확인을 요구하거나 간접 프롬프트 인젝션 감지 시 작업을 중단하는 안전 장치를 제공한다고 밝혔다.
- NVIDIA는 Nemotron 3 Ultra를 장시간 에이전트용 550B MoE 오픈 모델로 공개하며, 복잡한 에이전트 작업 비용을 최대 30% 낮추고 추론 속도를 최대 5배 높인다고 설명했다.
- 최신 arXiv 연구는 agent skill runtime enforcement, 시간 유효성을 가진 memory, 행동 단위 governance, GUI agent 경험 학습, 멀티모달 자기진화처럼 “에이전트가 실제로 행동할 때 무엇을 보장할 것인가”에 집중한다.

## 산업 뉴스와 정책 흐름

### 1) OpenAI: 업무의 단위가 대화에서 장시간 위임으로 이동한다

OpenAI의 `How agents are transforming work` 분석은 에이전트형 AI가 기존 챗봇과 다른 경제적 단위를 만든다는 점을 숫자로 보여준다. 챗봇 상호작용은 짧고 자기완결적인 경우가 많지만, 에이전트는 수분에서 수시간 동안 도구를 호출하고 환경과 상호작용하며 해결책을 반복적으로 찾아간다. OpenAI는 Codex 사용 패턴을 근거로, 사람들이 점점 더 긴 시간의 업무를 에이전트에게 맡기고 있다고 설명한다.

특히 흥미로운 수치는 비개발자 사용자의 증가다. Codex는 코딩 도구로 출발했지만, 법무, 재무, 리크루팅 같은 비기술 부서에서도 자동화, 데이터 변환, 도구 제작, 구조화 분석에 쓰이고 있다. OpenAI 내부에서는 2026년 6월 기준 Codex가 주간 출력 토큰의 대부분을 차지하고, 일부 사용자는 하루에 60시간이 넘는 Codex agent turns를 여러 병렬 작업으로 발생시킨다고 설명한다. 이는 AI 도구가 “질문에 답하는 보조자”에서 “병렬로 돌리는 업무 실행 단위”로 이동한다는 신호다.

이 변화는 기업의 AI 도입 판단도 바꾼다. 생산성은 더 이상 프롬프트 하나의 답변 품질만으로 측정되지 않는다. 에이전트가 긴 작업을 맡을수록 중요한 것은 권한, 실행 로그, 실패 복구, 중간 산출물 검증, 비용 통제, 병렬 작업 관리다. 오늘의 연구 흐름에서 VIGIL, MemStrata, institutional attestation 같은 논문이 두드러지는 것도 같은 이유다. 에이전트가 실제 업무를 수행할수록 모델 자체보다 실행 경계가 중요해진다.

### 2) OpenAI와 Anthropic: 국방 AI는 성능 경쟁이 아니라 레드라인 경쟁이 된다

OpenAI는 미국 Department of War와의 계약을 설명하며 세 가지 레드라인을 강조했다. 대규모 국내 감시 금지, 자율무기 시스템을 독립적으로 지휘하는 용도 금지, 사회 신용 같은 고위험 자동 의사결정 금지다. OpenAI는 배포가 클라우드 기반이며, 자체 안전 스택을 유지하고, 모델을 guardrails-off 형태로 제공하지 않으며, cleared personnel과 안전 연구자가 관여한다고 설명했다. 추가 업데이트에서는 미국인과 미국 국민에 대한 국내 감시, 상업적으로 취득한 식별 가능 정보의 감시 목적 사용, NSA 같은 정보기관 사용 제한을 더 명시했다고 밝혔다.

Anthropic도 미국 국방부 CDAO와 2년, 2억 달러 한도의 prototype other transaction agreement를 발표했다. Anthropic은 DOD 데이터에 맞춘 working prototype, adversarial AI misuse 예측, 기술 통찰과 운영 피드백 교환을 언급했다. Claude Gov 모델과 Claude for Enterprise, AWS 기반 공공 부문 배포, Lawrence Livermore National Laboratory와 Palantir를 통한 국방·과학 워크플로 사례도 함께 제시했다.

두 발표를 나란히 보면 frontier AI 기업의 국방 진입은 피할 수 없는 시장 확대이면서 동시에 신뢰 경쟁이다. 어떤 모델이 더 강한가보다 더 중요한 질문은 누가 어떤 배포 구조를 택하고, 어떤 사용을 계약과 기술로 금지하고, 어떤 로그와 검증 경로를 남기는가다. AI 회사는 이제 “모델 개발사”가 아니라 고위험 조직 환경에서 실행 권한을 배포하는 인프라 회사가 되고 있다.

### 3) Google Gemini 3.5 Flash: computer use가 기본 모델 기능으로 들어왔다

Google은 Gemini 3.5 Flash에 computer use를 내장했다고 밝혔다. 이전에는 별도 computer use 모델로 제공되던 기능이 이제 Gemini Flash 모델의 기본 built-in tool로 들어오면서, 개발자는 브라우저, 모바일, 데스크톱 환경을 보고 추론하고 행동하는 custom agent를 더 쉽게 만들 수 있다. Google은 이를 지속적인 소프트웨어 테스트, professional application 사이를 넘나드는 지식 노동 자동화, 장시간 enterprise automation에 연결한다.

안전 장치도 함께 강조됐다. Google은 computer use에 targeted adversarial training을 적용했고, 기업이 민감하거나 되돌릴 수 없는 행동에 명시적 사용자 확인을 요구할 수 있게 하며, 간접 프롬프트 인젝션이 식별되면 작업을 자동 중단하는 optional safeguard를 제공한다고 설명한다. 또한 sandboxing, human-in-the-loop verification, strict access controls와 함께 쓰라고 권고한다.

이 발표의 의미는 명확하다. AI 에이전트의 다음 표준 인터페이스는 텍스트 API만이 아니라 화면, 마우스, 키보드, 모바일 UI, 업무 앱 전체다. 이때 agent safety는 모델이 “나쁜 답변을 하지 않는가”가 아니라 “권한 있는 행동을 언제 멈추고 누구에게 확인받는가”의 문제다. computer use가 메인 모델에 들어왔다는 것은 GUI automation이 부가 기능이 아니라 에이전트 플랫폼의 핵심 기능이 됐다는 뜻이다.

### 4) NVIDIA Nemotron 3 Ultra: 장시간 에이전트는 오픈 모델과 보안 런타임을 함께 요구한다

NVIDIA는 GTC Taipei/COMPUTEX 업데이트에서 Nemotron 3 Ultra를 공개했다. 550B MoE 모델인 Nemotron 3 Ultra는 장시간 에이전트의 어려운 reasoning call, 수백 개 자료에 대한 synthesis, 대규모 constraint 검증, 긴 coding session의 architecture decision을 겨냥한다. NVIDIA는 이 모델이 최대 5배 빠른 추론과 복잡한 에이전트 작업 비용 최대 30% 절감을 제공한다고 설명한다.

중요한 것은 모델만 공개한 것이 아니라 생태계를 함께 내세웠다는 점이다. Nemotron 3 Ultra는 open weights, datasets, recipes를 제공하고, Hugging Face, ModelScope, OpenRouter, build.nvidia.com, NIM microservices, cloud partner를 통해 배포된다. 또한 NemoClaw blueprint와 연결돼 secure runtime, domain-specific skills, agent harness를 묶는다. NVIDIA는 speech recognition model과 multimodal content safety model도 함께 언급했다.

이는 오픈 모델 경쟁의 초점이 단순 benchmark score에서 agent deployment stack으로 옮겨가고 있음을 보여준다. 기업은 모델을 가져다 쓰는 것만으로는 부족하다. 장시간 에이전트에는 orchestration, runtime isolation, policy enforcement, domain skill, cost routing, content safety, 음성 인터페이스가 필요하다. NVIDIA가 Nemotron Coalition, Naver Cloud, Nous, Prime Intellect 같은 참여자를 언급한 것도, 에이전트 모델이 단일 회사 제품보다 연합 생태계에 가까워지고 있기 때문이다.

### 5) Anthropic Claude Corps: AI 확산은 노동시장 적응 프로그램이 된다

Anthropic의 Claude Corps는 AI 기업이 모델 접근성만이 아니라 사회적 적응 비용까지 직접 다루려는 사례다. Anthropic은 초기 1억 5천만 달러를 투입해 1,000명의 fellow에게 Claude 활용 교육을 제공하고, 미국 전역의 비영리 조직에 1년간 배치하겠다고 밝혔다. Fellow는 8만 5천 달러의 급여와 교육, 멘토링, Claude token budget을 받으며, 최소 400개 비영리 조직이 참여한다.

이 프로그램은 제품 발표라기보다 배포 전략이다. AI가 노동시장에 충격을 준다면, frontier AI 기업은 “모델을 출시했다”에서 멈출 수 없다. 실제 조직이 어떤 업무를 자동화하고, 어떤 역량을 배워야 하며, 어떤 윤리적 기준으로 현장 문제에 적용할지까지 다루게 된다. Claude Corps는 AI 리터러시가 개인 스킬이자 조직 역량이 되는 시점의 신호다.

## 최신 연구 포커스

### 1) VIGIL: 에이전트 스킬 정책은 문서가 아니라 실행 중 제약이어야 한다

`VIGIL: Runtime Enforcement of Behavioral Specifications in AI Agent Skills`는 에이전트가 third-party skill을 통해 파일, 커뮤니케이션 채널, cyber-physical device에 영향을 줄 때, 자연어 스펙만으로는 충분하지 않다고 지적한다. 스킬 설명에는 접근 권한, 공개 제한, 실행 권한, 사전 조건이 적혀 있지만, 실제 실행 중 이를 강제하는 구조가 없으면 정책은 문서에 머문다.

VIGIL은 agent-tool event trace를 behavioral policy와 operator-defined constraint, global rule에 맞춰 검사한다. temporal dependency, argument constraint, value-flow condition을 표현하는 policy language를 만들고, 이를 finite trace 위의 SMT constraint로 바꿔 위반을 탐지한다. 논문은 실제 LLM-agent run에서 95% 이상의 recall과 10% 미만의 false-positive rate를 보고한다. 에이전트 시대의 안전은 프롬프트에 “하지 말라”고 쓰는 것이 아니라, 실행 흔적과 도구 호출을 정책 언어로 검사하는 방향으로 간다.

### 2) MemStrata: 에이전트 memory에는 시간 유효성이 필요하다

`Temporal Validity in Retrieval Memory`는 RAG가 시간 개념을 갖지 못해 stale fact error를 만든다고 지적한다. 어떤 함수명이 바뀌거나 API가 재구성되면, RAG는 오래된 사실과 최신 사실을 모두 비슷한 embedding similarity로 검색한다. 그 결과 에이전트는 답을 회피하거나 이미 폐기된 값을 제시한다.

MemStrata는 사실의 subject, relation, object 구조를 이용해 모순되는 새 값이 들어오면 기존 값을 bi-temporal ledger에서 retire시키는 retrieval memory다. 논문은 RAG가 evolving knowledge에서 0.20-0.47 accuracy에 머무는 반면 MemStrata가 0.95-1.00 accuracy에 도달하고, stale-fact-error rate를 사실상 0에 가깝게 낮춘다고 보고한다. 장시간 에이전트가 조직의 문서, 코드베이스, API 변경 이력을 다룰수록 “무엇을 아는가”보다 “언제까지 참인가”가 중요해진다.

### 3) Governing Actions, Not Agents: 고위험 행동은 독립 증거로 승인돼야 한다

`Governing Actions, Not Agents`는 자율 에이전트를 통제하는 단위를 “agent itself”가 아니라 “consequential action”으로 바꿔야 한다고 주장한다. 인간 제도는 강력한 행위자를 reasoning 감시로만 통제하지 않고, 중요한 행동의 순간에 독립적으로 증명된 조건과 서명을 요구해 왔다. 논문은 이를 AI agent governance에 옮겨, 에이전트는 계획과 추론을 자유롭게 하되 고위험 행동 실행 권한은 갖지 않는 모델을 제안한다.

실행은 별도 authoritative source가 attestation한 precondition, declared intent에 cryptographically bound된 evidence, deterministic policy 평가를 통과해야 가능하다. 이 접근은 OpenAI와 Google이 말한 human confirmation, cloud-only deployment, sensitive action safeguard와 같은 산업 흐름과 맞닿아 있다. 에이전트 안전의 핵심은 모델 마음속을 완전히 읽는 것이 아니라, 실제 세계를 바꾸는 행동에 독립 검증 가능한 문턱을 세우는 것이다.

### 4) GUI agent와 computer use: 경험 탐색과 hindsight가 계획 능력을 키운다

`Empowering GUI Agents via Autonomous Experience Exploration and Hindsight Experience Utilization for Task Planning`은 멀티모달 웹 에이전트가 반복적인 GUI 작업을 돕기 위해서는 task planning이 핵심이라고 본다. 작은 오픈소스 MLLM은 비용과 프라이버시 측면에서 유리하지만, cross-website generalization과 planning이 약하다. 논문은 에이전트가 환경을 자율 탐색해 experience를 발견하고, hindsight experience로 high-level training data를 합성하는 PEEU 방법을 제안한다.

이 연구는 Google Gemini 3.5 Flash computer use 발표와 바로 연결된다. 모델이 화면을 보고 클릭할 수 있어도, 낮은 수준의 atomic skill만으로는 높은 수준의 계획 능력이 보장되지 않는다. 논문은 high-level task training이 OOD generalization에 더 중요하다고 보고하며, 7B 모델이 더 큰 Qwen2.5-VL-32B를 넘는 결과를 제시한다. GUI 에이전트 경쟁은 화면 인식보다 경험 축적과 계획 데이터의 품질 싸움으로 갈 가능성이 크다.

### 5) Self-evolving multimodal models: 멀티모달 모델은 스스로 질문하고 풀고 생성한다

`Ask, Solve, Generate`는 통합 멀티모달 모델이 사람 주석, preference label, 외부 reward model 없이 unlabeled image만으로 visual understanding과 image generation을 함께 개선할 수 있는지 묻는다. Proposer가 시각 질문을 만들고, Solver가 답하고 평가하며, Generator가 이미지를 합성하는 세 역할을 내부적으로 구성한다. self-derived consistency signal과 Solver Token Entropy, cycle-consistent captioning을 통해 이해와 생성을 묶는다.

같은 흐름에서 `Paying More Attention to Visual Tokens`는 self-evolving LMM이 답의 일관성만 맞추다가 실제 시각 증거를 덜 보는 visual under-conditioning 문제를 지적한다. VISE는 geometric invariance와 semantic invariance reward로 모델이 이미지의 시각 token에 더 의존하도록 유도한다. 멀티모달 AI의 다음 병목은 “이미지를 볼 수 있는가”가 아니라 “정말 이미지 증거에 근거해 답하고 생성하는가”다.

## 오늘의 관찰

오늘 뉴스와 논문을 관통하는 단어는 “실행 권한”이다. OpenAI의 Codex 분석은 에이전트가 이미 사람의 하루 업무 시간을 병렬로 흡수하는 방향으로 가고 있음을 보여준다. Google의 computer use는 그 에이전트가 실제 화면과 앱을 조작하는 단계로 들어왔음을 뜻한다. NVIDIA의 Nemotron 3 Ultra는 장시간 에이전트가 오픈 모델, 보안 런타임, 비용 최적화, 음성·안전 모델을 포함한 스택으로 배포돼야 한다는 산업적 답을 제시한다.

동시에 OpenAI와 Anthropic의 국방 AI 발표는 고위험 배포에서 기술 기업이 어떤 레드라인을 계약, 아키텍처, 사람의 개입, 안전 스택으로 구현할 것인지 묻는다. Anthropic의 Claude Corps는 AI의 이익을 사회적으로 확산하려면 교육과 현장 적용 프로그램이 필요하다는 점을 보여준다. AI 기업들은 이제 모델 카드와 API 문서만으로 설명되지 않는 책임을 갖게 됐다.

연구도 같은 방향이다. VIGIL은 스킬 정책을 실행 중 제약으로 바꾸고, MemStrata는 memory의 시간 유효성을 관리하며, institutional attestation은 고위험 행동 승인 구조를 모델 밖으로 뺀다. GUI agent 연구는 화면 조작 능력보다 경험과 계획 학습이 중요하다고 말하고, 멀티모달 자기진화 연구는 모델이 스스로 만든 보상 신호로도 발전할 수 있지만 시각 증거를 실제로 사용하게 만드는 제약이 필요하다고 말한다.

결국 2026년 중반의 AI 경쟁은 “더 똑똑한 모델”을 넘어 “더 오래 실행되고, 더 많은 권한을 갖고, 더 다양한 환경을 조작하는 시스템을 어떻게 검증할 것인가”로 이동하고 있다. 이 질문에 답하려면 모델 성능, 인프라 비용, 정책 언어, 실행 로그, cryptographic attestation, human confirmation, memory validity가 한꺼번에 설계돼야 한다. 에이전트 시대의 안전은 모델 내부의 성향만이 아니라, 행동이 현실로 나가는 마지막 문턱을 누가 어떻게 지키는가에 달려 있다.
