아키텍처 & 핵심 기능

🏗️ 아키텍처 특징
  • • 5:1 하이브리드 로컬/글로벌 어텐션 — 근거리 5토큰 로컬 + 글로벌 슬라이딩 윈도우
  • • GQA (Grouped Query Attention) — 메모리 효율적 어텐션
  • • MoE: 128 소형 전문가 + 1 공유 — 토큰당 8개 활성 (Llama 4는 16 대형 전문가)
  • • Proportional RoPE + Softcapping — 위치 임베딩 스케일링 + 값 클리핑
  • • KV Cache Sharing — MoE 전문가 간 KV 캐시 공유
⚡ 핵심 기능
  • • Thinking Mode — 4,000+ 토큰 단계별 추론 후 답변
  • • Function Calling — 네이티브 API 호출, JSON 구조화 출력
  • • 멀티모달 비전 — 가변 해상도 이미지/비디오, OCR, 차트 이해
  • • 네이티브 오디오 (E2B/E4B) — ASR+번역+추론 단일 모델
  • • 에이전트 워크플로우 — 시스템 명령, 도구 호출 자율 에이전트

결론 & 전망

Gemma 4는 단순한 업그레이드가 아닌 패러다임 전환이다. Apache 2.0 라이선스 전환, Gemini 3 기반 아키텍처, 네이티브 멀티모달, Thinking Mode, 에이전트 워크플로우 지원을 하나의 모델 패밀리에 통합했다. 특히 26B-A4B의 MoE 설계는 소비자 GPU에서도 프론티어급 성능을 경험하게 한다.
✅ 강점
  • 1. 수학·코딩 압도적 — AIME 89.2%, CF 2150
  • 2. MoE 효율성 — 3.8B 활성으로 97% 품질
  • 3. Apache 2.0 — 경쟁사보다 자유로운 상업 이용
  • 4. 엣지 오디오 — E2B에서 단일 모델 ASR+번역+추론
  • 5. 에이전트 네이티브 — Function Calling, JSON 내장
⚠ 한계
  • 1. 다국어 — Qwen 3.5 (201개 언어)이 여전히 우위
  • 2. 컨텍스트 — Llama 4의 1M+에는 미치지 못함
  • 3. 추론 속도 — MoE 모델의 tok/s 벤치마크 미확인
  • 4. 신생 모델 — 커뮤니티 파인튜닝 축적 필요
  • 5. GPQA/MMLU — Qwen 3.5가 근소 우세
