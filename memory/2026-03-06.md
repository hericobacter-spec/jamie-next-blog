## Home Assistant Telegram 패널/명령 (HA_BASE_URL/HA_TOKEN)
- 사용자 요청으로 Telegram에서 버튼만 눌러 제어 가능한 HA 패널을 복구/재구성.
- OpenClaw 로컬 플러그인 추가: `workspace/.openclaw/extensions/ha` (id: `ha`).
  - `/ha` 슬래시 커맨드 제공(LLM 우회 auto-reply command):
    - `toggle|on|off <entity_id>`
    - `state <entity_id>` (climate는 mode+현재온도+목표온도 표시로 개선)
    - `temp <climate_entity_id> <21-26>`
    - `open|close <valve_entity_id>`
- Telegram 인라인 버튼 패널(고정용 메시지) 구성:
  - 스위치: 거실1/거실2/복도1/복도2 켜기/끄기/토글, 일괄소등
  - 온도: 안방/거실/서재/세연이방 21~26 바로 선택(방 선택 2단계 제거)
  - 가스밸브: 열기/닫기/상태
- 실제 작동 확인 완료(사용자 확인).

## iMessage 수신 감지(imsg) 연동 시도
- 사용자 요구: iMessage 수신 감지(watch) + 발신(send), 이 Mac mini에서 로컬(Messages.app) 기반.
- GitHub repo: steipete/imsg 확인 후 로컬 빌드:
  - Xcode CLT + Swift 6.2.4 환경에서 `make build` 성공, `imsg` v0.5.0 생성.
  - sudo 불가로 `/Users/jamie/.local/bin/imsg`에 설치(사용자 PATH에 포함됨).
- OpenClaw 로컬 플러그인 초안 추가: `workspace/.openclaw/extensions/imsg` (id: `imsg`, `/imsg send|chats|history`)
  - Telegram 네이티브 커맨드 등록에서 "/imsg" 충돌 로그 발생(추후 커맨드명 변경 또는 텍스트/비네이티브 방식 검토 필요).
- 수신 워처(알림만) 구현 시도:
  - Python 스크립트: `workspace/imsg-watcher/imsg_watch_to_telegram.py`
  - launchd LaunchAgent: `~/Library/LaunchAgents/com.jamie.imsg-watcher.plist`
  - 최초 구현에서 launchd KeepAlive 재시작 루프 시 "시작됨" 알림이 반복 전송되어 사용자 불만 → 워처 즉시 중지.
- 스팸 방지 조치:
  - 시작 알림 비활성화.
  - 에러 텔레그램 알림은 10분에 1회로 제한.
  - LaunchAgent에 `ThrottleInterval=60` 추가.
  - 디버그 로그 `/tmp/imsg-watcher.debug.log`에 rc/상태 기록.
- 현재 관측:
  - `imsg chats`는 정상 동작.
  - launchd로 실행되는 워처는 `imsg watch`가 rc=1로 종료되며 stderr가 비어있는 케이스가 발생(권한/TCC 주체 차이 가능성 높음).
  - 사용자가 Full Disk Access를 부여했다고 보고함(터미널 vs launchd/python3/node 권한 주체 확인 필요).
